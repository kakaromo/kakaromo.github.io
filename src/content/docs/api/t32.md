## T32 Config CRUD

Base path: `/api/t32`

| Method | Path | 설명 |
|--------|------|------|
| GET | `/configs` | 전체 T32 설정 목록 (DTO, 서버 이름 해석 포함) |
| POST | `/configs` | T32 설정 추가 |
| PUT | `/configs/{id}` | T32 설정 수정 (비밀번호 빈 문자열이면 기존 값 유지) |
| DELETE | `/configs/{id}` | T32 설정 삭제 (cascade로 매핑도 삭제) |

### 요청 본문 (POST/PUT)

```json
{
  "serverGroupId": 1,
  "jtagServerId": 5,
  "jtagUsername": "",
  "jtagPassword": "",
  "t32PcId": 6,
  "t32PcUsername": "admin",
  "t32PcPassword": "secret",
  "jtagCommand": "cd T32linker && echo yes | python ./tmux_lib.py -n 1 -t {tentacle_num} -s {slot}",
  "jtagSuccessPattern": "Measured Voltage.*Register Value",
  "t32PortCheckCommand": "t32rem.exe localhost port=20000 system.mode.attach",
  "dumpCommand": "cmd /c C:\\T32\\dump.bat {branch_path} {result_path}",
  "fwCodeLinuxBase": "/appdata/samsung/OCTO_HEAD/FW_Code",
  "fwCodeWindowsBase": "F:\\FW_Code",
  "resultBasePath": "/appdata/samsung/OCTO_HEAD/T32_dump",
  "resultWindowsBasePath": "F:\\T32_dump",
  "description": "Lab A T32 설정",
  "enabled": true,
  "assignedServerIds": [3, 4, 5]
}
```

### 필드 설명

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `serverGroupId` | Long | O | Lab 서버 그룹 ID |
| `jtagServerId` | Long | O | TMUX 서버 ID (portal_servers) |
| `jtagUsername` | String | | TMUX 전용 SSH 계정 (비워두면 서버 기본) |
| `jtagPassword` | String | | TMUX 전용 SSH 비밀번호 (비워두면 서버 기본) |
| `t32PcId` | Long | O | T32 PC 서버 ID (portal_servers) |
| `t32PcUsername` | String | | T32 PC 전용 SSH 계정 (비워두면 서버 기본) |
| `t32PcPassword` | String | | T32 PC 전용 SSH 비밀번호 (비워두면 서버 기본) |
| `jtagCommand` | String | | JTAG 연결 명령어 (`{tentacle}`, `{tentacle_num}`, `{slot}`) |
| `jtagSuccessPattern` | String | | Step 1 성공 판정 정규식 |
| `t32PortCheckCommand` | String | | T32 Attach 확인 명령어 |
| `dumpCommand` | String | | Dump 실행 명령어 (`{tentacle}`, `{slot}`, `{result_path}`, `{branch_path}`) |
| `fwCodeLinuxBase` | String | | Linux FW 코드 기본 경로 |
| `fwCodeWindowsBase` | String | | Windows FW 코드 기본 경로 |
| `resultBasePath` | String | | Linux 결과 저장 기본 경로 |
| `resultWindowsBasePath` | String | | Windows 결과 저장 기본 경로 |
| `description` | String | | 메모 |
| `enabled` | boolean | O | 활성화 여부 |
| `assignedServerIds` | Long[] | | 담당 텐타클 서버 ID 목록 |

## T32 Dump 실행

Base path: `/api/t32/dump`

### Config 확인

```
GET /check?serverId={id}
```

해당 서버에 T32 설정이 있는지 확인합니다. 컨텍스트 메뉴 활성화 판단용.

```json
{ "available": true, "configId": 1 }
```

### Dump 실행 (SSE)

```
POST /execute
Content-Type: application/json
→ text/event-stream (timeout: 600초)
```

### 요청 본문

```json
{
  "serverId": 5,
  "tentacleName": "T10",
  "slotNumber": 1,
  "fw": "Savona_V8",
  "branchPath": "/appdata/samsung/OCTO_HEAD/FW_Code/Savona/Savona_V8_P00RC28",
  "setLocation": "T10-1",
  "testToolName": "randwrite",
  "testTrName": "Savona_V8_TLC_512Gb_512GB_P00RC28"
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| `serverId` | Long | 텐타클 서버 ID (T32Config 매핑 조회용) |
| `tentacleName` | String | 텐타클 이름 (T10 등) |
| `slotNumber` | int | 슬롯 번호 |
| `fw` | String | FW 버전 (미사용 시 빈 문자열) |
| `branchPath` | String | FW 소스 폴더 Linux 경로 (Windows로 자동 변환) |
| `setLocation` | String | 슬롯 위치 (결과 폴더명 조합용) |
| `testToolName` | String | 테스트 도구명 (결과 폴더명 조합용) |
| `testTrName` | String | TR 이름 (결과 폴더명 조합용) |

### 결과 폴더명 조합

```
{date}_{setLocation}_{testToolName}_{testTrName}
```

예시: `20260407_T10-1_randwrite_Savona_V8_TLC_512Gb_512GB_P00RC28`

빈 값은 건너뛰어 `20260407_T10-1`처럼 짧아질 수 있습니다.

### SSE 이벤트

| 이벤트 | 데이터 | 시점 |
|--------|--------|------|
| `step-start` | `{step: number, name: string}` | 각 Step 시작 |
| `step-output` | `{step: number, line: string}` | SSH stdout 라인 (실시간) |
| `step-done` | `{step: number, status: "success"\|"failed", output: string}` | Step 완료 또는 실패 |
| `dump-progress` | `{step: 3, core: string, phase: string, status: string}` | Step 3 Core별 진행 |
| `done` | `{success: boolean, resultPath: string, resultWindowsPath: string}` | 전체 완료 |
| `error` | `{message: string}` | 예외 오류 |

### Step별 성공 판정

| Step | 성공 조건 |
|------|-----------|
| 1 (JTAG) | `jtagSuccessPattern` 정규식이 stdout에 매칭 |
| 2 (Attach) | exitCode=0 AND stdout에 "Down" 미포함 |
| 3 (Dump) | exitCode=0 AND stdout에 "fail" 미포함 (대소문자 무시) |

### 타임아웃

| Step | 제한 시간 |
|------|-----------|
| Step 1 (JTAG) | 30초 |
| Step 2 (Attach) | 30초 |
| Step 3 (Dump) | 300초 (5분) |

## TC Pre-Command API

Base path: `/api/pre-commands`

### TC별 조회

```
GET /tc?source={source}&slotIndex={slotIndex}
```

```json
[
  { "tcId": 34, "preCommandId": 1, "preCommandName": "Init UFS" },
  { "tcId": 38, "preCommandId": 2, "preCommandName": "Hang for T32" }
]
```

### TC별 등록

```
POST /tc/assign
```

```json
{
  "preCommandId": 1,
  "source": "compatibility",
  "slotIndex": 3,
  "tcId": 34
}
```

### TC별 해제

```
POST /tc/unassign
```

```json
{
  "source": "compatibility",
  "slotIndex": 3,
  "tcId": 34
}
```

### 동작 규칙

- TC에 Pre-Command가 있으면 → 슬롯 Pre-Command 무시 (TC 우선)
- 슬롯 clear 시 → 해당 슬롯의 모든 TC Pre-Command 자동 삭제
- TC 삭제 시 → 해당 TC의 Pre-Command 자동 삭제
