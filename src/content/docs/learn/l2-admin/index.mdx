## 30초 요약

Admin 대시보드는 **18 탭 단일 페이지**(`admin/+page.svelte` 2652줄):

- **System**: Health · Connections · Cache · App Info · Config
- **Infrastructure**: Servers · Server Groups · Head TCP · VM Status
- **Users & Auth**: Users · Sessions · **Permission Requests**
- **Data/Tools**: Sets · Slots · UFS Info · Debug · Metadata · Bitbucket · T32
- **Menus**: 메뉴 가시성 토글

백엔드는 **`AdminController`**(618줄)가 단일 진입점. `/api/admin/*` 네임스페이스에 ~40개 엔드포인트.

**관리 UI 축 개척** — 기존 15 L2는 "한 기능"이었다면 Admin은 **"기능들을 관리하는 메타 UI"**. 다른 관리 UI 후보(Guacamole Admin, T32 Admin 등)와 패턴 비교 가능.

## 관리 UI 축의 특징

| 속성 | Admin 대시보드 | 기능 L2 (슬롯·벤치마크 등) |
|---|---|---|
| 타겟 | 다른 기능의 설정/상태 | 자기 도메인 |
| 주 독자 | 운영자/관리자 | 일반 사용자 |
| 실시간성 | 낮음 (폴링 위주) | 높음 (SSE 다수) |
| 복잡도 분산 | 한 파일에 집중(2652줄) | 파일당 중간 |
| 변경 빈도 | 낮음 | 중~높음 |

**"운영자가 다른 L2를 관리하는 화면"** 이 공통 정체성. 향후 Guacamole Admin, T32 Admin 이 같은 패턴이면 축 확정.
