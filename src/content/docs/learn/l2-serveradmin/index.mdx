## 30초 요약

Server Admin은 Admin 대시보드의 **"Servers / Server Groups" 탭**이 담당. 원격 접속 서버(SSH/RDP/VNC), Head TCP 서버, guacd 서버 등 **여러 프로토콜과 역할**을 한 테이블(`portal_servers`)에 통합 관리.

- **PortalServer**: 이름 · IP · 3 포트(ssh/rdp/vnc) · `connectionType`(비트마스크) · guacdHost/Port · visible
- **ServerGroup**: 서버를 논리적으로 묶음 (sortOrder로 정렬)
- **UI**: Admin 페이지 탭 — 그룹별 카드 뷰 + 서버별 편집 폼
- **연결 타입 비트**: SSH=1, RDP=2, VNC=4 — 한 서버가 여러 프로토콜 동시 지원

**관리 UI 축 확정 시도**: Admin 대시보드(첫 구성원)에 이어 Server Admin이 두 번째. 둘 다 관리 도메인이지만 **대상 범위**가 다름.

## PortalServer와 ServerGroup의 역할

| 엔티티 | 목적 |
|---|---|
| `portal_servers` | 개별 서버 — IP · 포트 · 프로토콜 · guacd 설정 |
| `portal_server_groups` | 서버 그룹 — 이름 + 정렬 순서 |

**"서버가 여러 L2에 쓰이는 공통 자원"**:

- 원격 L2: WebSocket 경유 guacd 연결
- Head TCP L2: 자체 TCP 프로토콜
- Log Browser L2: SSH 세션 (tentacle)
- Pre-Command L2: SSH 경유 adb
- T32 Dump L2: JSch × 2 서버

**한 서버 row = 여러 L2가 참조**. Admin UI가 이를 단일 진입점으로 관리.
