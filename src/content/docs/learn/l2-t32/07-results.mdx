## 30초 요약

Step 3 에서 만든 **result 폴더 ZIP** 을 Step 4 가 **MinIO 에 업로드**하고, 그 위치를 DB 한 행(`T32ResultArtifact`)으로 남깁니다. 나중에 호환성/성능 테스트 화면에서 "그때 그 dump" 를 다시 받을 수 있게.

- **objectKey** = `{testType}/{source}/{historyId|nohistory}/{resultDirName}.zip`
- **상태 전이** = `UPLOADING` → `COMPLETED` (또는 `FAILED`)
- **history 매칭** = `(testType, source, historyId)` **튜플** — 하드 FK 가 아니라 (cross-datasource)
- **보존** = `uploadedAt` 기준 **60일**, 매일 04:00 KST 에 S3 객체 + DB row 함께 삭제
- **로컬 결과는 보존** — 업로드 후에도 result 폴더/ZIP 을 지우지 않음 (완료 직후 다이얼로그가 그 폴더를 listdir)

## 조회 + 다운로드 (프론트)

```
GET /api/t32/results/latest?testType=COMPATIBILITY&source=INTERNAL&historyId=42
  → 최신 COMPLETED 아티팩트 1건 (없으면 404)
  → { bucket, objectKey, sizeBytes, resultDirName, status, uploadedAt }
  → 프론트가 bucket+objectKey 로 MinIO presigned 다운로드
```

다이얼로그의 **"S3에서 다운로드"** 버튼이 이 흐름입니다. 방금 끝난 dump 는 업로드가 완료될 때까지 잠깐 시간이 걸리므로 **3회 재시도**합니다. `historyId`+`testType` 컨텍스트가 있을 때만 버튼이 노출됩니다.

## cleanup — 매일 04:00 KST

```java
@Scheduled(cron = "0 0 4 * * *", zone = "Asia/Seoul")
public void runCleanup() {
    if (!enabled) return;                          // t32.result.cleanup-enabled
    var cutoff = LocalDateTime.now().minusDays(retentionDays);  // 기본 60
    for (var a : artifactRepository.findByUploadedAtBefore(cutoff)) {
        storage.deleteObject(a.getBucket(), a.getObjectKey());  // S3 먼저
        artifactRepository.delete(a);                            // DB row 같이
    }
}
```

**버킷 lifecycle 대신 스케줄러**를 쓰는 이유: S3 객체와 **DB row 를 동시에** 정리해야 둘이 어긋나지 않기 때문. Trace 의 `TraceExportCleanupTask` 와 같은 패턴입니다.

:::note[운영 메모 — DDL 수동 적용]
모든 프로파일이 `ddl-auto: validate` 라 신규 테이블 `portal_t32_result_artifacts`(및 `portal_t32_configs` 의 신규 컬럼 `t32_remote_host`/`t32_remote_port`, `core_scripts_json`, `t32_start_command`)는 `sql/add-*.sql` / `sql/alter-*.sql` 로 운영자가 **수동 적용**해야 합니다. 빠뜨리면 validate 가 부팅을 막습니다.
:::

## 직접 해보기

- [ ] dump 완료 후 DB `portal_t32_result_artifacts` 에 `status=COMPLETED` 행이 생기는지
- [ ] MinIO 콘솔에서 `t32-results/COMPATIBILITY/INTERNAL/{historyId}/...zip` 객체 확인
- [ ] MinIO 를 잠깐 꺼두고 dump → 행이 `FAILED` + errorMessage 인데 dump 는 success 로 끝나는지
- [ ] `retention-days` 를 0 으로 두고 cleanup 을 수동 실행 → S3·DB 가 같이 비는지
