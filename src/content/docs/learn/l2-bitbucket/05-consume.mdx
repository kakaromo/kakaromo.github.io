## 30초 요약

[T32 Dump L2](/learn/l2-t32/02-trigger/)가 이 모듈의 **두 경로를 소비**합니다.

1. **브랜치 목록** (`GET /api/bitbucket/repos/{id}/branches`) — 드롭다운에 DETECTED/DOWNLOADED 브랜치 표시
2. **인라인 다운로드** (`POST /api/bitbucket/branches/{branchId}/download` SSE) — DETECTED 상태 브랜치를 선택 시 진행률 카드로 다운로드

또한 **Admin Bitbucket 탭**(`AdminBitbucketTab.svelte`)이 직접 같은 API를 사용 — Repo CRUD와 수동 다운로드 관리.

## 컨트롤러 엔드포인트 요약

| Method | Path | 용도 |
|---|---|---|
| GET | `/api/bitbucket/repos` | Repo 전체 조회 |
| POST | `/api/bitbucket/repos` | Repo 생성 |
| PUT | `/api/bitbucket/repos/{id}` | Repo 수정 |
| DELETE | `/api/bitbucket/repos/{id}` | Repo 삭제 |
| GET | `/api/bitbucket/repos/{id}/branches` | 해당 Repo의 브랜치 이력 |
| POST | `/api/bitbucket/branches/{id}/download` | **DETECTED → 다운로드 시작 (SSE)** |
| POST | `/api/bitbucket/branches/{id}/delete-files` | 파일만 삭제 (status → DETECTED) |
| POST | `/api/bitbucket/repos/{id}/poll` | 수동 즉시 폴링 트리거 |
| POST | `/api/bitbucket/repos/{id}/test` | PAT·URL 연결 테스트 |

T32가 쓰는 건 **`GET /branches`와 `POST /download` 두 개**.

## T32 드롭다운 (의사코드)

```svelte
<!-- T32DumpDialog.svelte (부분) -->
<script lang="ts">
  import { onMount } from 'svelte';
  import { listBranchesOfAllRepos, downloadDetectedBranch } from '$lib/api/bitbucket';

  let branches = $state<BitbucketBranch[]>([]);
  let query = $state('');

  // 상태순 정렬: DOWNLOADED → DETECTED → DOWNLOADING → FAILED
  const statusOrder = { DOWNLOADED: 0, DETECTED: 1, DOWNLOADING: 2, FAILED: 3 };

  async function loadBitbucketBranches() {
    branches = await listBranchesOfAllRepos();
    branches.sort((a, b) => {
      return (statusOrder[a.status] ?? 9) - (statusOrder[b.status] ?? 9);
    });
  }

  const bbFiltered = $derived(
    branches.filter((b) => !query || b.branchName.includes(query))
  );

  onMount(loadBitbucketBranches);

  async function handleInlineDownload(branch: BitbucketBranch) {
    await downloadDetectedBranch(
      branch.id,
      (mb) => updateCard(branch.id, 'downloading', mb),
      () => { updateCard(branch.id, 'done'); loadBitbucketBranches(); },
      (err) => updateCard(branch.id, 'failed', err),
    );
  }
</script>

<input bind:value={query} placeholder="브랜치 검색" />

{#each bbFiltered as b (b.id)}
  <div class="row" class:downloaded={b.status === 'DOWNLOADED'}>
    <span>{b.branchName}</span>
    <span class="date">{formatDate(b.commitDate)}</span>
    {#if b.status === 'DOWNLOADED'}
      <button onclick={() => selectBranch(b)}>Select</button>
    {:else if b.status === 'DETECTED'}
      <button onclick={() => handleInlineDownload(b)}>Download</button>
    {:else if b.status === 'DOWNLOADING'}
      <span>⏳</span>
    {:else}
      <span title={b.errorMessage}>⚠ Failed</span>
    {/if}
  </div>
{/each}
```

핵심:

- **상태순 정렬** — 바로 쓸 수 있는 DOWNLOADED가 위, 실패는 아래
- **검색 필터** — 브랜치 많을 때 빠른 검색
- **DETECTED에만 Download 버튼** — 사용자가 필요한 것만 받음 (용량 관리)

## 프론트 SSE 핸들러

```ts
// lib/api/bitbucket.ts
export async function downloadDetectedBranch(
  branchId: number,
  onProgress?: (mb: number) => void,
  onDone?: () => void,
  onError?: (msg: string) => void,
) {
  const resp = await fetch(`/api/bitbucket/branches/${branchId}/download`, {
    method: 'POST',
  });
  const reader = resp.body!.getReader();
  const decoder = new TextDecoder();
  let buffer = '';

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    buffer += decoder.decode(value, { stream: true });

    // event: download-progress / extract-start / extract-done / error
    let idx;
    while ((idx = buffer.indexOf('\n\n')) >= 0) {
      const block = buffer.slice(0, idx);
      buffer = buffer.slice(idx + 2);
      const ev = parseBlock(block);
      if (ev.event === 'download-progress') onProgress?.(ev.data.mb);
      else if (ev.event === 'extract-done') onDone?.();
      else if (ev.event === 'error') onError?.(ev.data.message);
    }
  }
}
```

T32 L2·Agent L2의 SSE 수신과 같은 **fetch ReadableStream + 블록 파싱** 패턴. `EventSource`는 GET만 지원하므로 POST에는 이 방식.

## BitbucketDownloadCard — 플로팅 진행률

Admin UI는 다운로드 시 `BitbucketDownloadCard`를 화면 우하단에 띄웁니다.

```
┌────────────────────────────────┐
│ 📦 feature_fw-v1.2             │
│ [████████░░░░░]  23 MB / ~58 MB │
│ status: downloading             │
└────────────────────────────────┘
```

T32 Dump의 Stepper UI와 유사한 **실시간 플로팅 UI**. 다만 step이 아닌 **단일 바 + MB 카운터**라 구조가 훨씬 간단.

## 아홉 L2의 "다른 L2에서 소비" 관계

```
T32 Dump ──┬──→ Bitbucket (브랜치 드롭다운)
           └──→ Log Browser (결과 폴더 탐색 + ZIP)

Pre-Command ──→ (없음, 독립)

Agent 벤치마크 ──┬──→ Go Agent (gRPC)
                └──→ Log Browser (trace 파일)  ← 잠재적

Excel Export ──→ Log Browser (JSON 파일)  ← PerformanceResultDataService 내부
```

**Log Browser**와 **Bitbucket** 둘이 가장 **재사용 밀도 높은 레이어**. 두 L2가 있어야 T32 전체 워크플로우가 완성됨.

- **Metadata**: 5초 폴링 (슬롯 상태 감지가 핵심 — 반응성 요구)

같은 도구를 **도메인 주기에 맞춰 다르게 설정**하는 판단.
