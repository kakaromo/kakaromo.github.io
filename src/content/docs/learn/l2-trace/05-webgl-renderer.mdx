## 30초 요약

ECharts 는 50k~200k 포인트까지는 부드럽지만 500k 이상은 렌더 2~5 초 + 인터랙션이 끊깁니다. Deck.gl 경로는:

1. **ScatterplotLayer binary attribute mode** — `Float32Array` (positions) + `Uint8Array` (RGBA colors) 를 그대로 GPU 버퍼로 업로드, 1M 60fps
2. **축/범례/브러시는 SVG overlay** — Deck.gl 은 GPU 에 데이터만 올리고, UI 요소는 DOM 에서 따로 그림
3. **OrthographicView** 로 2D scatter — `target` + `zoom` 만 갱신하면 pan/zoom
4. **Feature flag** (`VITE_TRACE_RENDERER=deckgl`) + **dynamic import** — flag off 사용자는 deck.gl 번들 미다운로드

props 는 ECharts 경로와 **완전 동일** — 교체가 자유로움.

## 왜 binary attribute 인가

일반적 Deck.gl 사용법은 `{ data: [...rows], getPosition: d => [d.x, d.y] }` 입니다. 각 row 마다 함수 호출 + 새 배열 생성 — 1M 포인트에서 시작 지연이 큽니다.

**binary attribute mode** 는:

```ts
data: {
  length: N,
  attributes: {
    getPosition: { value: float32Positions, size: 2 },
    getFillColor: { value: uint8Colors, size: 4, normalized: false }
  }
}
```

`TypedArray` 를 그대로 GPU 버퍼로 복사 — **0-copy 업로드**. 반복적 re-render(zoom/pan)에서도 버퍼 재사용이라 매 프레임 재계산이 없습니다.

## 왜 SVG overlay 인가

Deck.gl 은 **GPU 에 데이터만** 올립니다. 축·틱·라벨·범례·브러시 사각형 같은 2D UI 를 WebGL 로 그리면 번거롭고 느립니다. 대신:

- **축 overlay**: SVG `<line>` + `<text>` 로 눈금·라벨
- **범례 overlay**: DOM `<button>` 리스트, cmd 별 on/off 토글
- **브러시 overlay**: pointer event 로 pixel 좌표 수집 → `xScale.invert` / `yScale.invert` 로 data 좌표 역변환

Deck canvas 와 SVG/DOM 이 같은 `container` 에 absolutely position 되어 **좌표만 맞으면** 자연스럽게 합쳐 보입니다.

## 차트 간 동기화

6개 차트(LBA/QD/CPU/DtoC/CtoD/CtoC) 는 `chartSync.svelte.ts` store 를 공유:

- `xDomain` — 시간 범위. 하나에서 zoom 하면 모두 같은 x 로
- `hiddenCmds` — 범례에서 cmd off 하면 모두에 반영
- `cpuMode` — CPU 차트만 쓰는 'cmd' / 'lba' 토글

**무한 반복 방지**: 자신이 방금 zoom 한 차트는 `suppressViewStateReset = true` 플래그로 다음 초기화를 건너뜀. 서버 재요청이 돌아와 `xDomainShared` 가 갱신된 뒤(400ms) 다시 허용.

## Feature flag + dynamic import

```ts
// routes/trace/+page.svelte (요약)
const useDeckgl = import.meta.env.VITE_TRACE_RENDERER === 'deckgl';

{#if useDeckgl}
  {#await import('./deckgl/TraceChartViewDeck.svelte') then M}
    <M.default {...props} />
  {/await}
{:else}
  <TraceChartView {...props} />
{/if}
```

- flag off 면 `deckgl/*` 번들이 아예 다운로드되지 않음 (build-time code split)
- ECharts 가 기본 — 50k 포인트에서 충분히 부드럽고, 축/범례/브러시가 기본 제공
- Deck.gl 은 opt-in — **500k 이상 시계열 밀도를 정확히 보고 싶을 때만** 의미가 있음

## 제약

- **축/범례/브러시 전부 직접 구현** — ECharts 기능을 그대로 복제해야 했기에 코드 양이 증가 (OrthographicView + SVG overlay 합계)
- **tooltip 미구현** — 1M 포인트에서 hover pick 은 느림. 필요하면 `pickable: true` + quadtree 스크리닝
- **GPU 메모리 한계** — 1M 60fps 는 안정적이지만 10M 은 장비 의존 (데스크톱 GPU 환경 필요)
