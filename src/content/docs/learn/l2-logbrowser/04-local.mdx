## 30초 요약

`LocalLogBrowserService`는 NFS/SMB로 포털 서버에 **마운트된 동일 파일 시스템**을 로컬 파일처럼 다룹니다.

- **탐색**: `java.nio.file.Files.list()`
- **검색**: `ProcessBuilder("rg", ...)` — 서버 로컬에서 직접 실행
- **보안**: `resolveLocalPath`가 **path traversal 방어** (경로가 base 밖으로 나가면 `SecurityException`)

SSH 구현과 반환 타입·정렬 규칙이 완전히 같아 **UI에서 두 모드를 구분 불가능** (의도된 설계).

## 왜 path traversal 방어가 필수인가

Local 모드는 **포털 서버 프로세스의 권한**으로 파일을 읽습니다. 사용자가 `../../../etc/passwd` 같은 경로를 보내면 마운트 범위 **밖**의 파일도 읽힐 수 있음.

```java
Path baseDir = Paths.get(mountPath, vmName).normalize();      // /mnt/T1
Path resolved = baseDir.resolve(path).normalize();             // /mnt/T1/../etc/passwd → /etc/passwd
if (!resolved.startsWith(baseDir)) throw new SecurityException;
```

- `normalize()`가 `..`을 해석해 실제 경로로 축소
- `startsWith(baseDir)`로 탈출 여부 감지

**SSH 모드는 SSH 서버의 로그인 사용자가 자동으로 격리**되므로 이 검사가 필요 없음. Local 모드에서만 직접 방어가 필요.

## HEAD 특수 분기

```java
if ("HEAD".equalsIgnoreCase(vmName)) {
    baseDir = Paths.get(headMountPath).normalize();       // /mnt/head/nas
} else {
    baseDir = Paths.get(mountPath, vmName).normalize();   // /mnt/T1
}
```

HEAD는 Tentacle VM이 아니라 **공용 NAS**를 가리킴 — 경로 구성이 다릅니다. 이 분기 하나로 인터페이스·컨트롤러는 "HEAD도 그냥 tentacle 중 하나"처럼 다룰 수 있음.

## 두 구현 요약 비교

| 작업 | SSH | Local |
|---|---|---|
| listFiles | `ChannelSftp.ls` | `Files.list` |
| download | `SftpInputStream`(래퍼) | `Files.newInputStream` |
| search | `ChannelExec rg` (shellEscape) | `ProcessBuilder rg` (인자 배열) |
| 인증 | JSch 세션 | 포털 서버 프로세스 권한 |
| 인젝션 방어 | shellEscape | ProcessBuilder 인자 |
| 경로 방어 | SSH 서버 로그인 격리 | resolveLocalPath + startsWith |
| 세션 캐시 | 5분 TTL | 없음 |
