## 한 문장 요약

**MOVE (Mobile OS Validation Eco-system)** 는 Samsung 내부에서 모바일 OS·스토리지를 검증하는 사람들이 테스트 장비·실제 디바이스·로그·리포트를 한 브라우저에서 다루기 위한 포털입니다.

## 풀려는 문제

전통적으로 검증 파이프라인은 여러 도구의 조합이었습니다 — 장비 화면을 보려면 원격 데스크톱, 로그는 SSH, 벤치마크는 스크립트, 결과 리포트는 엑셀. 도구 사이 이동과 수동 조작이 많을수록 **재현 가능성**, **협업 속도**, **검증 깊이**가 떨어집니다.

MOVE는 이 작업을 한 곳에 모으고, 기계가 할 수 있는 일(반복 실행, 데이터 수집, 차트 생성, 리포트 내보내기)은 기계에게 맡기는 것을 지향합니다.

## 주요 외부 연동

MOVE는 스스로 모든 것을 하지 않고, 역할이 분명한 **외부 서비스·장비**를 조립합니다.

- **Head 서버** — 실제 UFS 하드웨어를 제어하는 C 기반 서버. MOVE는 TCP로 슬롯 상태를 읽고 명령을 보냅니다.
- **Tentacle VM / 실제 장비** — 테스트 대상 장비. SSH·RDP로 원격 접속하거나 로그 파일을 가져옵니다.
- **Go Agent** (:50051) — Android 디바이스 앞단에 붙는 gRPC 서버. 벤치마크·trace·시나리오·매크로 실행 담당.
- **Go Excel Service** (:50052) — Go + `excelize/v2` 기반 gRPC. 성능 결과를 네이티브 Excel 차트와 함께 내보냅니다.
- **Rust Trace Service** (:50053) — `arrow` + `parquet` + `rayon` 기반 gRPC. 대용량 I/O trace 를 parquet 으로 변환하고 Arrow IPC 로 차트 데이터를 바이너리 전송.
- **ADFS** — Samsung 사내 SSO. OIDC Hybrid Flow 수동 구현.

이 외부 경계를 이해하면 MOVE 자체 코드가 왜 이런 구조인지 설명이 쉬워집니다. 다음 페이지의 [시스템 아키텍처](/learn/l1-overview/architecture/)에서 블록 간 연결을 한눈에 볼 수 있습니다.

## 이 프로젝트가 아닌 것

- 일반 사용자용 SaaS 제품이 아닙니다 — 내부망에서만 동작하며 ADFS(SSO)로 인증합니다.
- 자체적으로 장비를 제어하는 저수준 드라이버가 아닙니다 — Head/Agent 같은 전용 서비스에 위임합니다.
- 범용 BI/시각화 도구가 아닙니다 — 검증 워크플로우에 특화된 차트·테이블·내보내기만 제공합니다.
