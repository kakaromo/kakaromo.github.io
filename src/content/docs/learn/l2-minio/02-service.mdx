`MinioStorageService`는 **SDK를 감싸는 얇은 서비스 레이어** (150줄). 컨트롤러가 MinioClient를 직접 쓰지 않고 이 서비스를 통해 호출하는 구조.

**9 메서드**:

| 영역 | 메서드 |
|---|---|
| 버킷 | `listBuckets` / `createBucket` / `deleteBucket` |
| 탐색 | `listObjects` (1 depth) / `listObjectsRecursive` (전체) |

## listObjects vs listObjectsRecursive — 왜 나눴는가

두 함수가 거의 같아 보이지만 **용도가 완전히 다름**:

| 함수 | 사용처 | 파라미터 핵심 |
|---|---|---|
| `listObjects` | UI 폴더 탐색 | `delimiter="/"` — 1 depth |
| `listObjectsRecursive` | ZIP 다운로드 · 배치 처리 | `recursive(true)` — 전체 |

**delimiter 있으면** MinIO가 prefix 매칭 결과에서 공통 prefix(`foo/bar/...`)를 폴더로 집약. **없으면** 원본 경로 그대로 모두 반환. 한 함수로 묶으면 분기 코드가 호출자마다 필요 → 나누는 게 더 깔끔.

## InputStream 반환 패턴 — 누가 close?

`downloadObject`는 `InputStream`을 돌려주고 끝. **close 책임은 호출자**.

- 컨트롤러 (`MinioController.downloadObject`): `InputStreamResource`로 감싸 Spring이 응답 후 close
- 폴더 ZIP (`downloadFolder`): `try-with-resources`로 각 항목 소비 후 close
- Agent gRPC 업로드 경로: Go Agent가 보낸 스트림을 직접 `uploadObject`에 전달

**SDK가 close 책임을 호출자에게 넘김** → 래퍼 없이 자원 관리 투명. Log Browser의 `SftpInputStream` 래퍼와 다른 접근.

## createFolder의 0 bytes 마커

```java
minioClient.putObject(
    PutObjectArgs.builder()
        .object(path)                                   // "foo/bar/"
        .stream(InputStream.nullInputStream(), 0, -1)   // 0 bytes
        .build());
```

S3/MinIO는 **진짜 디렉토리 개념이 없음**. 폴더는 "같은 prefix를 가진 객체 집합"의 추상. 빈 폴더를 UI에 표시하려면 `"foo/bar/"`로 끝나는 **0 bytes 객체**를 마커로 둠. `listObjectsRecursive`는 이 마커를 **`endsWith("/")`** 로 걸러냄.

## 9 메서드 전체 테이블

| 메서드 | 반환 | 내부 SDK |
|---|---|---|
| `listBuckets` | `List<String>` | `listBuckets()` |
| `createBucket` | void | `bucketExists` + `makeBucket` |
| `deleteBucket` | void | `removeBucket` |
| `listObjects` | `List<Map>` | `listObjects` + delimiter="/" |
| `listObjectsRecursive` | `List<Map>` | `listObjects` + recursive(true) |
| `deleteObject` | void | `removeObject` |
| `createFolder` | void | `putObject` (0 bytes) |
| `downloadObject` | `InputStream` | `getObject` |
| `statObject` | `StatObjectResponse` | `statObject` |
| `uploadObject` | void | `putObject` (stream) |
