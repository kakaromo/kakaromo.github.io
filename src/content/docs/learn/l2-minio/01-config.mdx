## 30초 요약

- **`MinioProperties`** — yaml `minio.*` 바인딩 (endpoint · port · useSsl · accessKey · secretKey · publicEndpoint · presignExpirySeconds · partSizeBytes · region)
- **`MinioConfig`** — **3개 빈** 등록:
  - `MinioClient` — MinIO Java SDK, 일반 CRUD/스트림 업로드용
  - `S3Client` — AWS S3 SDK, multipart upload (init/complete/abort) 직접 호출용
  - `S3Presigner` — Presigned URL 발급 (브라우저가 nginx 경유로 MinIO 직접 접근)

두 파일 합쳐 **97줄** (Properties 31 + Config 66). 두 SDK 를 같이 쓰는 이유는 MinIO Java SDK 가 multipart upload 의 일부 RPC (CreateMultipartUpload presigned 등) 를 직접 노출하지 않아 AWS S3 SDK 로 보완.

## 왜 이렇게 짧은가

**SDK의 고수준 추상화 덕분**. `io.minio:minio 8.5.14`가:

- HTTP 클라이언트
- 자격 증명 서명
- XML 응답 파싱
- 오류 처리

를 모두 내부에 숨겨서 포털은 **"버킷/오브젝트 이름 + 스트림"** 수준에서만 상호작용하면 됩니다.

비교: Bitbucket L2의 `BitbucketApiClient`는 **직접 HttpClient로** HTTP 호출·JSON 파싱·SSL 설정까지 작성해야 했음. MinIO는 "이미 만들어진 SDK"가 있어 Client 설정이 이렇게 짧음.

## prod/dev 환경 분리 전략

현재 MOVE의 구조:

| 환경 | MinIO 위치 | 접속 |
|---|---|---|
| dev | 단일 VM (192.168.x.x:9000) | 공유 dev 자원 |
| prod | 전용 인스턴스 | 고정 IP |

yaml 프로파일 기반(`application-dev.yaml` / `application-prod.yaml`)으로 분기. **설정 키는 동일**, 값만 다름 — 코드 경로는 환경 무관.

## 3 Bean · 2 서비스 구조

```
MinioConfig (@Configuration)
    │ @Bean × 3
    ├──▶ MinioClient    (싱글턴) ──▶ MinioStorageService  (일반 CRUD + 스트림)
    │                                      │
    │                                      ▼
    │                              MinioController / MinioUploadController
    │
    ├──▶ S3Client       (싱글턴) ──┐
    └──▶ S3Presigner    (싱글턴) ──┴──▶ S3PresignService   (multipart presigned URL)
                                                 │
                                                 ▼
                                       (TraceJobController 등이 활용)
```

| 빈 | 사용처 | 패턴 |
|---|---|---|
| `MinioClient` | `MinioStorageService` 11 메서드 | 서버측 CRUD / 폴더 ZIP / 스트림 업로드 |
| `S3Client` | `S3PresignService.createMultipartUpload / completeMultipartUpload / abortMultipartUpload` | 서버측 multipart 메타 RPC |
| `S3Presigner` | `S3PresignService.presignAllParts / presignUploadPart / presignSinglePut` | 브라우저가 직접 PUT 할 URL 발급 |

Spring DI 가 자동 연결하므로 작성하는 건 필드 선언 + `@RequiredArgsConstructor` 생성자뿐.

업로드 경로 두 가지:
- **legacy** (`MinioUploadController` form-data) — 작은 파일, 서버를 통과
- **multipart presigned** (`S3PresignService` + 호출 컨트롤러) — 큰 파일, 브라우저 → nginx → MinIO 직결 (서버 메모리/네트워크 우회)
