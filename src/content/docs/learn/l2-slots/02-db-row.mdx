## 30초 요약

`HeadConnection`은 **MOVE가 연결할 Head 서버 한 대**를 표현하는 JPA 엔티티입니다. 테이블 이름은 `portal_head_connections`. 관리자 화면에서 행을 추가하면 단계 3의 TCP 클라이언트가 그 행을 읽고 연결을 시작합니다.

## 한 행의 구성

| 컬럼 | 의미 |
|---|---|
| `id` | 자동 증가 PK |
| `name` | 사람이 읽는 이름 (UK). SSE `source` 파라미터로도 쓰임 |
| `head_type` | 0=호환성, 1=성능 |
| `host` | Head 서버의 호스트/IP |
| `port_suffix` | outbound 포트 2자리 (실제 포트 = 10000 + suffix) |
| `listen_port_suffix` | inbound 포트 2자리 |
| `enabled` | 활성 여부 (`false`면 기동 시 스킵) |
| `test_mode` | 테스트 인스턴스 전용 여부 |
| `created_at / updated_at` | `@PrePersist` / `@PreUpdate`로 자동 갱신 |

`headType`은 0과 1 두 값이 현재 사용되며, 향후 확장(2, 3…)도 고려된 정수 필드입니다. 클래스 상수 `TYPE_COMPATIBILITY`, `TYPE_PERFORMANCE`로 참조하는 것이 약속입니다.

## <ConceptPopover client:visible term="@Entity" concept="spring/at-entity"><p>JPA 엔티티 선언 어노테이션. 클래스를 DB 테이블에 매핑된 엔티티로 등록합니다. 테이블 이름은 기본이 클래스명(snake_case)이며 <code>@Table(name=...)</code>로 재정의할 수 있습니다.</p></ConceptPopover> 와 함께 쓰는 Lombok

`HeadConnection`은 Lombok의 `@Data @Builder @NoArgsConstructor @AllArgsConstructor` 4개를 함께 씁니다.

- `@Data` — 게터/세터/equals/hashCode/toString 자동
- `@Builder` — `HeadConnection.builder().name("foo")...build()` 패턴 활성화
- `@NoArgsConstructor` — JPA 요구사항(기본 생성자 필요)
- `@AllArgsConstructor` — `@Builder`가 내부적으로 사용

이 네 개 조합은 MOVE 엔티티 전반의 표준입니다. 다른 엔티티를 읽을 때도 같은 패턴이 보일 것입니다.

## 왜 연결 상태·에러는 이 엔티티에 없는가

`HeadConnection`은 **정적 설정**만 담습니다. 현재 연결되어 있는지(`connected`), 마지막 에러(`error`)는 런타임 상태라 **단계 5의 `HeadSlotStateStore`** 가 메모리로 관리합니다. 재시작하면 런타임 상태는 사라지지만 설정은 DB에 남습니다.

이 분리 덕분에 DB 쓰기 빈도가 낮게 유지됩니다. 단계 6의 SSE 페이로드에서 연결 상태가 함께 내려가는 이유도 여기 있습니다 — 설정과 런타임 상태를 한 화면에서 합쳐 보여주기 위함입니다.

## 직접 해보기
