## 왜 7단계인가

MOVE는 슬롯 모니터링을 **계층 분리**로 설계했습니다. 단계가 짧아지거나 합쳐지면 다음 문제가 생깁니다.

- **파싱 로직과 전송 로직을 섞으면** — 프로토콜이 바뀔 때마다 컨트롤러를 고쳐야 합니다.
- **저장 없이 바로 푸시하면** — 재연결한 클라이언트가 "처음부터 다시" 받을 수 없습니다.
- **DB 조회를 SSE 주기마다 하면** — 1초×슬롯수×클라이언트수의 부하가 DB에 갑니다.

각 단계는 "무엇이 바뀌면 어디가 흔들리는지"가 명확하도록 분리되어 있습니다. 본문을 읽으면서 **"이 책임을 다른 단계로 옮기면 무엇이 불편해질까?"** 를 스스로 물어보면 설계 감을 잡는 데 도움이 됩니다.

## 학습 순서와 예상 소요 시간

| 단계 | 제목 | 성격 | 시간 |
|---|---|---|---|
| 1 | 하드웨어 · yaml 설정 | 읽기 | 5분 |
| 2 | DB 한 행 (HeadConnection) | 읽기 | 5분 |
| 3 | HeadTcpClient — 듀얼 소켓 | 코드 투어 | 15분 |
| 4 | HeadMessageParser | 코드 투어 | 10분 |
| 5 | HeadSlotStateStore | 코드 투어 | 10분 |
| 6 | HeadSseController | 코드 투어 | 15분 |
| 7 | headSlotStore (프론트) | 코드 투어 | 10분 |
| 99 | 리팩토링 여정 | 심화 | 옵션 |

총 약 **1시간 10분** + "직접 해보기"로 실제 연결까지 해보면 2시간 내. 본문만 훑을 거면 각 페이지 상단 요약과 다이어그램만 봐도 40분이면 끝납니다.
