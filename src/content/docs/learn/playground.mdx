<CodeTour
  client:visible
  language="java"
  source="HeadSseController.java"
  height={420}
  code={`@RestController
@RequiredArgsConstructor
public class HeadSseController {

    private final HeadSlotStateStore stateStore;
    private final List<SseEmitter> emitters = new CopyOnWriteArrayList<>();
    private long lastVersion = -1;

    @GetMapping("/api/head/slots/stream")
    public SseEmitter stream() {
        SseEmitter emitter = new SseEmitter(0L);
        emitter.onCompletion(() -> emitters.remove(emitter));
        emitter.onTimeout(() -> emitters.remove(emitter));
        emitters.add(emitter);

        // 초기 스냅샷 전송
        var snapshot = stateStore.snapshot();
        trySend(emitter, "init", snapshot);
        return emitter;
    }

    @Scheduled(fixedRate = 1000)
    public void pushDiff() {
        long version = stateStore.version();
        if (version == lastVersion) return;
        var diff = stateStore.diffSince(lastVersion);
        lastVersion = version;
        for (SseEmitter emitter : emitters) {
            trySend(emitter, "update", diff);
        }
    }

    private void trySend(SseEmitter emitter, String event, Object payload) {
        try {
            emitter.send(SseEmitter.event().name(event)
                .data(payload, MediaType.APPLICATION_JSON));
        } catch (IOException e) {
            emitters.remove(emitter);
        }
    }
}`}

## 3. ConceptPopover — 인라인 용어 팝오버

Spring Boot의 <ConceptPopover client:visible term="@Scheduled" concept="spring/at-scheduled">
  **주기 실행 어노테이션.** `fixedRate`는 작업 시작 시각 기준, `fixedDelay`는 종료 시각 기준으로 다음 실행을 예약합니다. `@EnableScheduling`이 필요하며 MOVE에서는 `PortalApplication`에 설정되어 있습니다.
</ConceptPopover>는 1초 주기 push에 사용되고, 그 결과를 프론트엔드의 <ConceptPopover client:visible term="$state" concept="svelte/state">
  **Svelte 5의 반응형 상태 rune.** 할당·배열 push 등 변이를 추적하며, deep proxy로 감싸집니다. 단순 참조 보관이 필요하면 `$state.raw`를 사용하세요.
</ConceptPopover>로 저장해 UI에 반영합니다.
