## 30초 요약

BinMapper는 **C struct 정의문과 바이너리 파일**을 받아 필드별로 해석해주는 사내 개발도구.

1. 사용자가 C 스타일 struct 정의를 텍스트 또는 파일로 제공
2. **Lexer**가 토큰(STRUCT, IDENT, LBRACE, …)으로 분해
3. **Parser**가 재귀 하강으로 `StructDefinition` IR 생성 (pragma pack / typedef / union / enum / bitfield 처리)
4. **BinaryReaderService**가 struct 크기·정렬·offset 계산 후 `ByteBuffer`로 타입별 읽기
5. `MappedResult` 트리를 프론트로 반환 → Table / Hex+Struct / JSON 3뷰로 표시

**다른 L2와 다른 축** — 이전 12개가 "시스템 내 어떤 흐름"이었다면 BinMapper는 **컴파일러 파이프라인을 축약한 엔진** 한 덩어리. 독립 devtool로 설치됨.

## 왜 "파이프라인 엔진"인가

지금까지의 L2는 Spring 컨트롤러 + 서비스 + 프론트 store 조합. BinMapper도 같은 외피를 쓰지만 **본질은 작은 컴파일러**:

```
┌─────────┐    Lexer     ┌──────────┐    Parser    ┌─────────┐    Mapper   ┌─────────┐
│ 입력 텍스트 │ ──────────→  │  Token   │  ──────────→  │ StructDef│ ──────────→ │ MappedRes│
│  (C src) │              │  Stream  │              │   IR    │             │  (트리)  │
└─────────┘              └──────────┘              └─────────┘             └─────────┘
```

- **4단계 unidirectional 파이프라인** — 한 방향, side effect 없음
- **상태 없음** — 요청마다 새 Lexer·Parser 인스턴스(DB에는 PredefinedStruct 사전만)
- **도메인 검증** — Java 표준이 아닌 "우리가 자주 쓰는 C struct 패턴만 지원"

## 앞 12 L2와의 대조

| 축 | 대부분의 L2 | Auth | TestDB | **BinMapper** |
|---|---|---|---|---|
| 주 소비자 | 다수 L2 | 모든 요청 | 6 L2 | **독립 devtool — 다른 L2 의존 없음** |
| 트리거 | 다양 | 자동(인터셉터) | 외부 tentacle + 읽기 | **사용자 업로드 단발** |
| 실시간 자원 | 일부 | 없음 | Reparse만 | **없음** (요청-응답 한 번) |
| 상태 머신 | 각 L2별 | 세션 | RUNNING/PASS/FAIL | **없음 (stateless 계산)** |
| 설계 영감 | 웹 시스템 흐름 | OAuth2 | ORM + FS | **컴파일러 구조 (Lexer/Parser/IR/Codegen)** |

**"웹이지만 내용은 컴파일러"** 라는 점이 BinMapper의 유일성.

## 지원 범위의 의도적 좁힘

C struct 정의 전부를 파싱하려면 clang/gcc 프론트엔드가 필요. MOVE는 "자주 쓰는 부분 집합"만 지원:

| 지원 | 미지원 |
|---|---|
| 기본 int/float/char/bool | 사용자 typedef 체인 완전 전개 |
| `#pragma pack(N)` | `__attribute__((aligned(N)))` 정밀 제어 |
| 고정 크기 배열 `x[10]` | 가변 배열 / VLA |
| 중첩 struct · union | 함수 포인터 / 콜백 타입 |
| bitfield (단순화) | bitfield 저장 단위 내 bit 위치 추적 |
| enum ↔ 값 매핑 | 매크로(`#define`) 치환 |

**의도된 제약**: 실제 기업 bin 파일(UFS descriptor, ELF 헤더, 트레이스 프레임)은 위 지원 범위에 들어감. 못 다루면 `StructInput.svelte`에서 사용자가 손으로 고침.
