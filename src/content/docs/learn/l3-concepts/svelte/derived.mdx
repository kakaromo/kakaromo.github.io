## 이 프로젝트에서의 용례

- **필터링·정렬**: `$derived(store.slots.filter((s) => s.connection >= 1))`
- **카운터·집계**: `$derived(slots.length)`, `$derived(slots.reduce(...))`
- **복잡한 가공**: `$derived.by(() => { /* 여러 줄 로직 */ return result; })`

slots 페이지에는 이런 파생 값이 수십 개 있습니다. 원본 `$state`가 바뀔 때 자동으로 재계산.

## 최소 예시

```svelte
<script lang="ts">
  let slots = $state<Slot[]>([]);
  const connectedCount = $derived(slots.filter(s => s.connected).length);

  // 복잡한 식: $derived.by
  const summary = $derived.by(() => {
    const byState = new Map<string, number>();
    for (const s of slots) {
      byState.set(s.state, (byState.get(s.state) ?? 0) + 1);
    }
    return Array.from(byState.entries());
  });
</script>

<p>연결: {connectedCount} / {slots.length}</p>
```

## 자동 의존성 추적

`$derived`는 본문 안에서 읽는 모든 `$state`를 **실행 시점에 추적**합니다. 따로 dependency list를 적을 필요가 없습니다.

```ts
let a = $state(1);
let b = $state(2);
let toggle = $state(true);

const sum = $derived(toggle ? a : b);
// toggle=true → a 변경 시 재계산
// toggle=false → b 변경 시 재계산
// 의존성이 동적으로 바뀐다.
```

## 지연 계산 (lazy)

`$derived`는 **누가 읽을 때만** 계산됩니다. 비싼 식이어도 화면에 표시되지 않으면 돌지 않습니다. 읽은 뒤 결과를 캐시하므로 같은 프레임에서 여러 번 읽어도 1회만 계산.

## `$derived.by` — 다중 라인

한 줄 식으로 표현할 수 없는 경우 `.by(() => ...)` 블록 형태를 씁니다. 타입도 반환값으로 자연스럽게 추론됩니다.

```ts
const stats = $derived.by(() => {
  if (slots.length === 0) return null;
  // 여러 통계 계산...
  return { min, max, avg };
});
```

## 주의할 점

- **파생 값은 읽기 전용**. 재할당하면 컴파일 에러.
- **부작용을 넣지 마세요**. 네트워크 호출·DOM 변경은 `$effect`로. `$derived`는 순수 계산만.
- **deep equality는 해주지 않음**. 계산 결과 객체가 새로 만들어지면 "바뀌었다"고 본다. 필요하면 memoize 패턴을 직접.
