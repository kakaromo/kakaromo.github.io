## 30초 요약

`MetadataDialog.svelte`는 **멀티 슬롯 탭 + 타입 선택 + 3종 차트 전환**(히트맵/라인/산점도)이 한 다이얼로그에 모인 컴포넌트입니다. 수집된 JSON 배열을 그대로 받아:

1. `flattenObject` — 중첩 객체를 `a.b.c` 점 표기 경로로 평탄화
2. `classifyKeys` — 평탄화된 키를 `number` / `string` / `array` / `object`로 분류
3. **array 키**는 히트맵, **number 키**는 라인·산점도

ECharts 하나로 세 차트를 모두 커버합니다.

## 컴포넌트 구조 (의사코드)

```svelte
<!-- MetadataDialog.svelte -->
<script lang="ts">
  import PerfChart from '$lib/components/PerfChart.svelte';
  import { fetchTypesForTr, fetchSlotMetadata, fetchSlotStatus } from '$lib/api/metadata';

  interface SlotTarget {
    tentacleName: string;
    slotNumber: number;
    trId?: number;
    headType: number;
  }
  interface Props {
    slots: SlotTarget[];
    onClose: () => void;
  }
  let { slots, onClose }: Props = $props();

  let activeSlotIdx = $state(0);
  let types = $state<UfsMetadataType[]>([]);
  let activeTypeKey = $state<string | null>(null);
  let entries = $state<Array<Record<string, unknown>>>([]);
  let monitoring = $state(false);

  const activeSlot = $derived(slots[activeSlotIdx]);

  // 슬롯 바뀌면 type 목록 재로드
  $effect(() => {
    (async () => {
      types = await fetchTypesForTr(activeSlot.trId, activeSlot.headType);
      const status = await fetchSlotStatus(activeSlot.tentacleName, activeSlot.slotNumber);
      monitoring = status.monitoring;
      activeTypeKey = types[0]?.typeKey ?? null;
    })();
  });

  // 타입 바뀌면 데이터 로드
  $effect(() => {
    if (!activeTypeKey) return;
    (async () => {
      entries = await fetchSlotMetadata(
        activeSlot.tentacleName, activeSlot.slotNumber, activeTypeKey,
      );
    })();
  });

  // 30초마다 폴링 (monitoring 중일 때만)
  $effect(() => {
    if (!monitoring || !activeTypeKey) return;
    const id = setInterval(async () => {
      entries = await fetchSlotMetadata(
        activeSlot.tentacleName, activeSlot.slotNumber, activeTypeKey,
      );
    }, 30_000);
    return () => clearInterval(id);
  });

  // flatten + classify
  const flatEntries = $derived(entries.map(flattenObject));
  const { numberKeys, arrayKeys } = $derived(classifyKeys(flatEntries));
</script>
```

세 개의 `$effect` — 슬롯 변경, 타입 변경, 폴링 — 각각 독립된 라이프사이클. `$effect` 패턴이 **3개 실시간 모듈 전반에서 재사용**되는 것을 다시 확인할 수 있습니다.

## 히트맵 option 핵심

```ts
function buildHeatmapOption(entries: FlatEntry[], arrayKey: string) {
  const times = entries.map((e) => e.time);
  const arrays = entries.map((e) => e[arrayKey] as number[]);
  const maxLen = Math.max(...arrays.map((a) => a.length));
  const indices = Array.from({ length: maxLen }, (_, i) => i);

  const data: [number, number, number][] = [];
  let min = Infinity, max = -Infinity;
  arrays.forEach((arr, t) => {
    arr.forEach((v, i) => {
      data.push([t, i, v]);
      if (v < min) min = v;
      if (v > max) max = v;
    });
  });

  return {
    xAxis: { type: 'category', data: times.map(String), name: 'time (s)' },
    yAxis: { type: 'category', data: indices.map(String), name: 'index' },
    visualMap: { min, max, calculable: true, orient: 'horizontal', bottom: 0 },
    series: [{ type: 'heatmap', data }],
  };
}
```

핵심 아이디어는 **`[t, i, v]` 3-tuple의 평탄한 배열**. ECharts가 이 포맷을 그대로 히트맵으로 렌더.

## 라인 / 산점도 option

```ts
function buildLineOption(entries: FlatEntry[], keys: string[]) {
  return {
    xAxis: { type: 'value', name: 'time (s)' },
    yAxis: { type: 'value' },
    series: keys.map((key) => ({
      name: key,
      type: 'line',          // or 'scatter'
      data: entries.map((e) => [e.time, e[key]]),
      showSymbol: false,
    })),
  };
}
```

**같은 entries 배열에서 `numberKeys` 몇 개를 골라 series로 변환**하면 라인 차트. `type: 'line'`을 `'scatter'`로 바꾸면 산점도. delta 토글은 각 value를 `(v - prev)`로 치환.

## <ConceptPopover client:visible term="$effect" concept="svelte/effect"><p>Svelte 5의 부작용 rune. 반응형 의존성이 바뀔 때 실행되고 cleanup 함수를 return해 이전 실행을 정리합니다.</p></ConceptPopover> 3개 분리의 이점

하나의 큰 `$effect`로 모두 처리할 수도 있지만, 셋으로 나누면:

- **의존성이 명시적** — 슬롯 바뀔 때 types 재로드, 타입 바뀔 때 entries 재로드, monitoring일 때만 폴링
- **cleanup 범위 최소화** — 폴링 cleanup이 타입 로드까지 영향 안 줌
- **각 단계가 독립적으로 실패** — types 로드 실패해도 이후 로직이 보호됨

**"한 $effect에 여러 책임을 넣지 않는다"** 는 Svelte 5 코드에서 자주 보이는 원칙.

## 내보내기 — JSON / Excel

대시보드 우상단 버튼으로:

- **JSON** — `entries` 전체를 다운로드. 원본 그대로 외부 분석 도구에 넘김.
- **Excel** — `flatEntries`에서 **number 필드만** 추려 `.xlsx`. 히트맵은 제외(2차원 배열은 Excel 표로 표현이 어색).

## 네 L2의 프론트 실시간 패턴 요약

| 축 | 슬롯 | 벤치마크 | 원격 | **Metadata** |
|---|---|---|---|---|
| 전송 | EventSource | EventSource | WebSocket | **HTTP 폴링 30s** |
| cleanup | `eventSource.close()` | `eventSource.close()` | `client.disconnect()` | **`clearInterval()`** |
| 실시간성 | 1초 | 수초 | 프레임 | **분** |
| 공통 | 모두 `$effect` + 리소스 + cleanup |

**$effect가 전송 방식을 추상화**합니다. EventSource, WebSocket, setInterval 어느 것이든 "`$effect` 안에서 획득하고 return에서 해제"라는 패턴이 동일.
