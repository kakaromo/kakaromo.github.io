## 30초 요약

t32api 는 **이벤트를 push 해주지 않습니다.** 그래서 t32remote 는 직접 **풀(pull)** 합니다:

- **Poller** (50ms 루프) — `GetState`/`GetPracticeState`/`GetMessage`/`STATE.TARGET()` 을 주기적으로 읽어 **변화가 있을 때만** 이벤트로 만든다. 동시에 **연결 헬스체크**(연속 5회 실패면 끊김 판정).
- **Broker** — non-blocking pub/sub. poller 가 publish 하면 진행 중인 스트리밍 RPC 구독자에게 fan-out. **느린 구독자는 드롭**(poller 를 절대 안 막음).
- **runWithProgress** — `ExecuteCommand`/`RunScript`/`Go` 의 공통 엔진. broker 구독 + worker goroutine + 이벤트 포워딩.

## Poller가 끌어오는 것들

| 풀 대상 | → 이벤트 | 비고 |
|---|---|---|
| `GetState()` | `StateChange` | run-state 변화. **연결 헬스체크 겸함**(연속 5회 실패=끊김). |
| `GetPracticeState()` | `DialogOpened` | 다이얼로그 진입 감지 → 정책 적용 트리거. |
| `GetMessage()` | `AreaOutput` | DLL 이 "최신 메시지"만 주므로 변경분만. |
| `PROGRESS_PCT`/`PROGRESS_LABEL` | `UserProgress` | .cmm 이 PRACTICE 변수로 진행률을 기록하는 규약. |
| `STATE.TARGET()` | `AreaOutput`/`ErrorEvent` | 타겟 결함 감지(기본 on, `T32REMOTE_POLLER_TARGET=0` 로 off). |

기본 간격 `defaultInterval = 50ms`, `T32REMOTE_POLL_MS` 로 조정. (패키지 doc-comment 는 "150ms" 라고 적혀 있지만 **코드 기본값은 50ms** — 문서 잔재.)

## 빌드 — 호스트 stub vs Windows 크로스 컴파일

cgo + DLL 이라 빌드가 두 갈래입니다.

```bash
# 1) 호스트(맥/리눅스) — stub 백엔드, 컴파일·테스트 sanity
make build-host          # CGO_ENABLED=0 → bin/t32remote-host, bin/t32cli-host
make test                # go test ./...  (stub로 핸들러 검증)

# 2) Windows 크로스 컴파일 — 진짜 DLL 백엔드
make build-win           # CGO_ENABLED=1 GOOS=windows GOARCH=amd64 CC=x86_64-w64-mingw32-gcc
make dist                # build-win + exe·examples·README를 zip으로

# (Windows 네이티브 빌드는 build.ps1 -Target all)
```

- **import 라이브러리** `third_party/t32/libt32api64.a` 는 `t32api64.def` 에서 `dlltool` 로 만든 **스텁 import lib** — 링크 타임엔 함수 이름만 필요하고, 진짜 DLL 은 **런타임에 Windows 호스트에서** 로드됩니다.
- 그래서 빌드 머신엔 DLL 이 없어도 되고, **실행 머신**(PowerView PC)에만 있으면 됩니다.

## Broker가 느린 구독자를 드롭하는 이유

`Broker.Publish` 는 **non-blocking** 입니다. 구독자 채널이 꽉 차 있으면 그 이벤트를 **버립니다.** 왜?

- poller 는 50ms 마다 돌아야 하는데, 한 느린 구독자(예: 네트워크가 느린 클라이언트) 때문에 **poller 전체가 막히면** 상태 감지·헬스체크가 멈춥니다.
- 진행률·AREA 는 "최신이 중요"한 데이터라 몇 건 누락돼도 치명적이지 않습니다(상태/완료/에러는 별도로 보장).

**"전체 정확성보다 전체 가용성"** 을 택한 트레이드오프입니다.
