## 30초 요약

t32remote 는 **두 바이너리**를 만드는 Go 모듈입니다:

- **`t32remote`** — gRPC 서버 데몬. `-addr`(기본 `:50551`) 하나만 받는다. PowerView 와 같은 Windows PC 에서 상주.
- **`t32cli`** — 디버깅용 CLI 클라이언트. `attach`/`exec`/`capture`/`eval` 등 서브커맨드로 데몬을 수동 조작.

서버는 cgo 백엔드(Windows)나 stub 백엔드(그 외)로 빌드되며, **단일 PowerView 세션**을 **worker goroutine 하나**로 직렬 제어하고, **50ms poller** 가 상태를 풀어 `ProgressEvent` 로 흘려보냅니다.

| 패키지 | 역할 |
|---|---|
| `internal/server` | gRPC 핸들러. 도메인별 파일(`handlers_lifecycle`/`_run`/`_read`/`_dialog`/`_window`/`_breakpoint`/`_exec`/`_oscmd`). |
| `internal/session` | **단일 attach** 수명 관리(mutex). 멱등 Attach, 끊김 감지. |
| `internal/t32` | 저수준 TRACE32 바인딩. `API` 인터페이스 + cgo/stub 구현 + `Worker` 직렬화. |
| `internal/poller` | 50ms 루프로 상태·AREA·진행률·타겟 폴 → 이벤트. 연결 헬스체크도 겸함. |
| `internal/progress` | `Broker` — non-blocking pub/sub. 느린 구독자는 드롭(poller 가 안 막히게). |
| `internal/dialog` | `DIALOG.SET` 정책(헤더 매치 → 값 입력 → 버튼 클릭). |
| `internal/window` | 순수 PRACTICE 명령 빌더(`WinPOS … : Register.view` 등). |

## TRACE32 측 사전 설정

t32remote 가 붙으려면 PowerView 의 `config.t32` 에 RCL(원격 제어) 채널이 켜져 있어야 합니다:

```text
RCL=NETASSIST
PORT=20000
PACKLEN=1024
```

그리고 `t32api64.dll` 이 실행 경로에서 찾아져야 합니다(`C:\T32\bin\windows64\` 에서 실행하거나 PATH 추가, 또는 exe 옆에 복사). PowerView 는 **콘솔/RDP 세션에 GUI 로 떠 있어야** USB 디버그 모듈을 봅니다 — 그래서 Portal 이 Step 1·2 사이에 PsExec 로 세션 안에 재시작하는 것([L2 T32 Dump 3](/learn/l2-t32/03-steps/)).

:::caution[포트 표기 불일치]
문서/예제마다 attach 기본 포트가 <strong>20000</strong>(TRACE32 config 예) 과 <strong>20002</strong>(클라이언트/서버 기본값) 로 엇갈립니다. Portal 은 <code>t32PortCheckCommand</code> 필드의 숫자를 RCL 포트로 넘기고(없으면 20000), <code>config.t32</code> 의 <code>PORT=</code> 와 맞춰야 합니다.
:::
