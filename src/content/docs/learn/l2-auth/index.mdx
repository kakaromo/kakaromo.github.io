## 이 모듈에서 배울 것

"사용자가 ADFS SSO로 로그인해서, **어떤 메뉴를 볼 수 있고 어떤 API를 호출할 수 있는지** 결정되기까지 — 모든 단계가 어떻게 연결되는가?"

열한 번째 L2. 앞 10 L2와 결정적으로 다릅니다:

- 앞 10 L2는 **기능 모듈** (슬롯, 벤치마크, Excel 등)
- 이 L2는 **크로스-컷 레이어** — 10 L2가 모두 의존

`portal.auth.disabled: false` 로 전환하는 순간 모든 요청이 이 모듈을 거치게 됩니다.

## 왜 이 L2가 마지막인가

앞 10 L2를 다 본 뒤에 읽어야 **의미가 살아남**.

- `action:agent:benchmark` 가 뭐 하는 권한인지 → Agent L2를 본 뒤
- `menu:slots` 가 감추는 기능이 뭔지 → 슬롯 L2를 본 뒤
- Head 접근 제한이 의미 있는 이유 → 슬롯 L2의 SSE source 파라미터를 본 뒤

**기능 이해 후에야 권한 설계가 이해됨**. 이 순서가 중요합니다.

## 앞 10 L2와의 대조 — 크로스-컷의 특징

| 축 | 기능 L2 (슬롯·벤치마크·…) | **Auth L2** |
|---|---|---|
| 의존 방향 | 다른 L2·레이어 의존 | **모든 L2가 이 모듈에 의존** |
| 엔드포인트 소비 주체 | 사용자·Admin·Agent | **모든 요청이 통과** |
| 비활성화 | 기능 토글 (enabled=false) | **`portal.auth.disabled=true`로 전체 우회** |
| 오류 응답 | 도메인별 (404·400·500) | **403 (권한 없음) 통합** |
| 테스트 환경 | 각 기능 독립 | **disabled 모드가 모든 기능을 뚫음** |

**Auth가 작동 안 하면 모든 L2가 작동 안 함**, 반대로 **Auth를 끄면 모든 L2가 통과**. 이 양극성이 설계의 핵심.

## 학습 순서와 예상 시간

| 단계 | 제목 | 성격 | 시간 |
|---|---|---|---|
| 1 | ADFS Hybrid Flow 시작 | 코드 투어 | 10분 |
| 2 | 콜백 + 세션 생성 | 코드 투어 | 15분 |
| 3 | 17 권한 정의 (메뉴 + 액션) | 코드 투어 | 10분 |
| 4 | URL 인터셉터 매칭 | 코드 투어 | 12분 |
| 5 | Head 접근 + 권한 요청 | 의사코드 | 10분 |

총 약 **57분**. 단계 2가 가장 복잡 — id_token 파싱·claims 추출·사용자 자동 생성·세션 저장 + 타임아웃.

## 세 가지 실행 모드

```
                ┌────────────────────────────────────────────┐
                │ portal.auth.disabled (Boolean)             │
                ├────────────────────────────────────────────┤
true (local dev)│ 모든 /api 통과, 가상 Developer 사용자 주입 │
                ├────────────────────────────────────────────┤
false + adfs off│ AuthController.me()가 401 반환, 로컬 로그인 │
                ├────────────────────────────────────────────┤
false + adfs on │ 실 ADFS 플로우, HttpSession, 인터셉터 체크  │
                └────────────────────────────────────────────┘
```

로컬 개발·CI·프로덕션이 **같은 코드로 세 모드**를 지원. 각 모드의 분기 지점이 이 L2 전반에 흩어져 있음.

## 새로 등장할 L3 개념

**없음**. 앞 L2에서 본 `@Configuration`·`@RestController`·`@Entity`·`HandlerInterceptor`(Spring MVC)가 전부. 세션 API(`HttpSession`)는 Servlet 표준.
