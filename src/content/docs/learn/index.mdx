## 신규 개발자 온보딩 트랙 (4시간)

처음 합류한 분에게 권장하는 순서입니다. 각 단계 끝에서 다음으로 자연스럽게 이어지도록 골랐습니다.

1. **L1 — 도메인 이해** ([무엇을 만드는가](/learn/l1-overview/domain/) → [시스템 아키텍처](/learn/l1-overview/architecture/) → [기술 스택](/learn/l1-overview/tech-stack/) → [주요 데이터 흐름 4선](/learn/l1-overview/key-flows/)) · 약 30분
2. **L2 첫 모듈** — [슬롯 모니터링](/learn/l2-slots/) 7단계 · 약 60분. Head TCP·SSE·StateStore 같은 핵심 패턴이 모두 등장
3. **L2 두 번째 — 자기 업무에 가까운 한 모듈** ([19종 비교 표](/learn/l2-compare/)에서 골라잡기) · 약 60분
4. **L3 — 처음 본 개념 정리** — L2를 읽으며 hover popover로 보였던 개념 중 헷갈렸던 것을 [L3 개념 사전](/learn/l3-concepts/spring/at-scheduled/)에서 짧게 복습 · 약 30분
5. **L2 비교 페이지로 한 번 더** — [19종 비교](/learn/l2-compare/)를 다시 보면, 처음에 보지 않았던 축들이 갑자기 의미 있게 들어옵니다 · 약 30분

작업 도중 막히면 해당 L2의 "직접 해보기" 체크리스트나 페이지 하단 "Edit page" 링크를 활용하세요.

## 이 가이드는 누구를 위한 것인가

- Samsung 내부의 신규 개발자
- Java/Spring 경험은 있지만 MOVE 포털 또는 Svelte 5가 처음인 분
- 한 번 작업했지만 몇 달 뒤 돌아와 다시 맥락을 맞춰야 하는 분

언어 기초 전반을 다루지 않습니다. **이 프로젝트에서 실제 쓰인 것만** 설명합니다. 더 깊게 들어가고 싶을 때는 각 개념 페이지 하단의 공식 문서 링크를 따라가세요.

## 구조

- **L1 — 프로젝트 전체상**: 도메인, 시스템 아키텍처, 기술 스택, 주요 데이터 흐름 4선
- **L2 — 기능 모듈 19종 + Trace 엔진 deep-dive**: 같은 템플릿(N단계 여정)으로 작성. [전체 카탈로그](/learn/l1-overview/domain/#l2-카탈로그-19-모듈)
- **L3 — 언어/프레임워크 개념**: Spring 4 / Svelte 5 / Columnar Data / Rust async / WebGL — L2에서 등장한 용어의 hover popover가 연결되는 사전

## L2를 여러 개 읽은 뒤에는

L2 모듈을 여러 개 훑었다면 [**L2 19종 비교**](/learn/l2-compare/)에서 축별 패턴과 갈림길을 한 페이지로 확인하세요. 새 L2를 쓸 때의 체크리스트로도 쓸 수 있습니다.
