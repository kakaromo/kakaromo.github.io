파일 스토리지는 MinIO S3 호환 스토리지를 웹에서 직접 탐색하고 관리할 수 있는 기능입니다. 테스트 관련 파일, FW 바이너리, 문서 등을 업로드하고 공유할 수 있습니다. 페이지 경로: `/storage`

## 버킷 관리

### 버킷 생성

**[+ 버킷 생성]** 버튼을 클릭하고 버킷 이름을 입력하여 새 버킷을 생성합니다.

### 버킷 삭제

빈 버킷을 삭제합니다.

:::caution
파일이 포함된 버킷은 삭제할 수 없습니다. 먼저 버킷 내 모든 파일을 삭제한 후 버킷을 삭제하세요.
:::

### 가시성 토글

버킷의 표시/숨김을 토글합니다. 숨김 처리된 버킷은 목록에서 보이지 않지만 삭제되지는 않습니다.

## 파일 탐색

### Breadcrumb 네비게이션

현재 위치가 Breadcrumb(경로 탐색 바)으로 표시됩니다. 각 경로 요소를 클릭하여 상위 디렉토리로 이동할 수 있습니다.

### 폴더 생성

현재 위치에서 새 폴더를 생성합니다. 폴더명을 입력하고 생성 버튼을 클릭합니다.

## 파일 업로드

:::caution
**2GB 초과 파일은 압축 형식으로만 업로드 가능합니다.** 2GB를 초과하는 파일은 `.zip`, `.gz`, `.tar`, `.7z`, `.rar` 등 압축 포맷으로 압축한 후 업로드하세요. 압축하지 않은 2GB 초과 파일은 업로드가 거부됩니다.
:::

### 드래그 앤 드롭

파일을 브라우저 영역에 드래그 앤 드롭하여 업로드합니다. 업로드 진행 상황이 프로그레스 바로 표시되며, 진행 중인 업로드를 취소할 수 있습니다.

### 다중 파일 배치 업로드

여러 파일을 동시에 선택하거나 드래그하여 배치 업로드할 수 있습니다. 큐 기반으로 순차 처리되며, 각 파일의 진행 상황이 개별적으로 표시됩니다.

### 폴더 업로드

**[Upload Folder]** 버튼을 클릭하면 로컬 폴더를 선택할 수 있습니다. 선택한 폴더의 **하위 디렉토리 구조가 그대로 유지**되며, 모든 파일이 순차적으로 업로드됩니다.

예시: 로컬 폴더 `mydir/` 선택 시:
```
mydir/
├── file1.txt
└── sub/
    └── file2.txt
```

MinIO에 저장되는 경로 (현재 위치가 `docs/`인 경우):
```
docs/mydir/file1.txt
docs/mydir/sub/file2.txt
```

:::note
폴더 업로드는 브라우저의 `webkitdirectory` API를 사용합니다. Chrome, Edge, Firefox 등 최신 브라우저에서 지원됩니다.
:::

## 파일 다운로드

파일 행의 다운로드 버튼을 클릭하여 개별 파일을 다운로드합니다. 다운로드 진행 상황이 프로그레스 바로 표시되며, 진행 중인 다운로드를 취소할 수 있습니다. 다중 파일 다운로드 시 큐 기반으로 순차 처리됩니다.

### 폴더 다운로드

폴더 이름 옆의 **다운로드 아이콘(↓)**을 클릭하면, 폴더 내 모든 파일을 재귀적으로 조회하여 **하나씩 순차 다운로드**합니다.

- 폴더 내 하위 폴더의 파일도 모두 포함됩니다
- 기존 다운로드 큐와 동일한 프로그레스 바가 표시됩니다
- 체크박스로 폴더를 선택한 후 **일괄 다운로드**해도 폴더 내 파일이 자동으로 포함됩니다

## 파일 삭제

파일 행의 삭제 버튼을 클릭하여 개별 파일을 삭제합니다.

## 체크박스 다중 선택

| 기능 | 설명 |
|------|------|
| **개별 선택** | 파일 행의 체크박스 클릭 |
| **전체 선택** | 헤더의 체크박스로 전체 선택/해제 |
| **일괄 다운로드** | 선택된 파일을 한 번에 다운로드 |
| **일괄 삭제** | 선택된 파일을 한 번에 삭제 |

:::tip
대용량 파일 업로드 시 네트워크 상태에 따라 시간이 걸릴 수 있습니다. 프로그레스 바로 진행 상황을 확인하세요.
:::
