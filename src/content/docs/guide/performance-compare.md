---
title: 성능 비교
description: 여러 성능 테스트 결과를 Chart Overlay, Side-by-Side, Delta Table로 비교하는 방법
---

성능 비교 기능은 여러 History의 성능 결과를 나란히 놓고 비교할 수 있는 기능입니다. FW 버전 간 성능 차이, 설정 변경에 따른 영향 등을 분석할 때 유용합니다.

## 비교 항목 선택

### CompareToggleCell

Performance History 목록에서 각 행의 비교 토글 셀을 클릭하면 해당 항목이 비교 대상에 추가됩니다.

### Chip Bar

선택된 항목들이 화면 상단의 Chip Bar에 표시됩니다. 각 Chip을 클릭하여 항목을 제거할 수 있습니다.

:::note
비교를 위해 최소 **2개** 이상의 항목을 선택해야 합니다.
:::

비교 항목을 선택한 후 비교 페이지로 이동합니다.

**페이지 경로**: `/testdb/performance/compare?ids=42,58`

URL의 `ids` 파라미터에 쉼표로 구분된 History ID가 포함됩니다.

## 3가지 뷰 모드

비교 페이지에서는 세 가지 뷰 모드를 제공합니다.

### Chart Overlay

선택한 모든 항목의 차트를 하나의 차트 위에 오버레이하여 표시합니다. **Baseline**으로 지정된 항목이 굵은 선으로 강조되며, 나머지 항목은 반투명하게 표시됩니다.

- 동일한 축과 스케일에서 직접 비교 가능
- 색상으로 각 항목 구분
- 마우스 휠로 줌 인/아웃

### Side-by-Side

각 항목의 차트를 개별 카드로 **가로 나란히** 배치합니다. 카드들이 수평으로 나열되어 직관적인 시각 비교가 가능하며, 항목이 많을 경우 가로 스크롤됩니다. 모든 카드는 기본 펼침 상태로 표시됩니다.

### Delta Table

Baseline 대비 다른 항목들의 차이를 수치와 백분율로 표시합니다.

| 컬럼 | 설명 |
|------|------|
| Metric | 측정 항목명 |
| Baseline | 기준 값 |
| Compare | 비교 대상 값 |
| Delta | 절대 차이 (Compare - Baseline) |
| Delta % | 백분율 차이 |

양수(성능 향상)는 녹색, 음수(성능 저하)는 빨간색으로 표시됩니다.

## Baseline 선택 및 변경

비교 페이지에서 임의의 항목을 **Baseline**으로 지정할 수 있습니다. Baseline은 Chart Overlay에서 강조 표시되고, Delta Table에서 기준 값으로 사용됩니다.

Baseline을 변경하려면 해당 항목의 Baseline 버튼을 클릭합니다.

## 항목 제거

비교 페이지에서 개별 항목을 제거할 수 있습니다. 단, **최소 2개 이상** 유지해야 하므로 2개일 때는 제거가 비활성화됩니다.

## 다른 FW 비교 추가

TR 상세 페이지에서 **다른 FW와 비교** 기능을 통해 다른 TR의 History를 비교 대상에 추가할 수 있습니다. TC가 고정된 상태에서 다른 FW의 동일 TC 결과를 검색합니다.

선택한 FW에 해당 TC의 결과가 없을 경우, **"이 FW에 해당 TC의 결과가 없습니다"** 안내 메시지가 표시되며, **다른 FW 선택** 버튼으로 돌아갈 수 있습니다.

:::tip
비교 기능은 동일한 parserId를 가진 History끼리 사용할 때 가장 의미 있는 결과를 얻을 수 있습니다.
:::
