title: Binary Struct Mapper

---

Binary Struct Mapper(BinMapper)는 C/C++ 구조체 정의를 기반으로 바이너리 데이터를 사람이 읽을 수 있는 형태로 매핑하는 개발 도구입니다. 페이지 경로: `/devtools/bin-mapper`

## BinMapper란

UFS 개발/디버깅 중 바이너리 덤프를 분석해야 할 때, 구조체 정의를 입력하면 바이너리 데이터의 각 필드를 자동으로 파싱하여 보여줍니다. 헥스 에디터와 구조체 정의를 결합한 분석 도구입니다.

## 바이너리 입력

바이너리 데이터를 두 가지 방법으로 입력할 수 있습니다:

| 방법 | 설명 |
|------|------|
| **파일 업로드** | 로컬 바이너리 파일을 업로드 |
| **서버 경로** | 서버에 있는 바이너리 파일의 경로 입력 |

## 구조체 입력

매핑에 사용할 C/C++ 구조체를 세 가지 방법으로 입력합니다:

| 방법 | 설명 |
|------|------|
| **텍스트 입력** | 구조체 코드를 직접 텍스트로 입력 |
| **헤더 파일 업로드** | `.h` 파일을 업로드 |
| **Predefined DB 선택** | 미리 등록된 구조체 목록에서 선택 |

## 엔디언 설정

바이너리 데이터의 바이트 순서를 설정합니다:

| 옵션 | 설명 |
|------|------|
| **Big Endian** | 빅 엔디언 (네트워크 바이트 순서) |
| **Little Endian** | 리틀 엔디언 (x86/ARM 기본) |
| **Auto** | 자동 감지 시도 |

## 3가지 뷰 모드

매핑 결과를 세 가지 뷰로 확인할 수 있습니다.

### Table 뷰

구조체 필드를 재귀적 트리 구조로 표시합니다. nested struct도 접기/펴기가 가능하며, 각 필드의 오프셋, 크기, 값이 표시됩니다.

### Hex+Struct 뷰

ImHex 스타일의 바이트별 색상 동기화 뷰입니다. 왼쪽에 헥스 덤프, 오른쪽에 구조체 필드가 표시되며, 필드를 선택하면 해당 바이트가 동일한 색상으로 하이라이트됩니다.

:::tip
Hex+Struct 뷰는 바이너리 데이터와 구조체 필드의 대응 관계를 시각적으로 파악할 때 가장 유용합니다.
:::

### JSON 뷰

매핑 결과를 JSON 형식으로 표시합니다. 프로그래밍 방식으로 결과를 활용하거나 복사할 때 유용합니다.

## Predefined Struct 관리

자주 사용하는 구조체를 미리 등록해두고 재사용할 수 있습니다.

- **등록**: 구조체 이름과 코드를 입력하여 저장
- **수정**: 기존 구조체 정의 변경
- **삭제**: 더 이상 필요 없는 구조체 제거

## 지원 기능

BinMapper는 다양한 C/C++ 문법을 지원합니다:

| 기능 | 설명 |
|------|------|
| **typedef** | `typedef struct` 등 타입 별칭 |
| **nested struct** | 구조체 내부 구조체 |
| **pragma pack** | `#pragma pack(1)` 등 패킹 지시자 |
| **배열** | 고정 크기 배열 (`uint8_t data[16]`) |
| **BOOL8 / BOOL32** | 8비트/32비트 불리언 타입 |
| **ASCII 힌트** | 바이트 배열을 문자열로 해석 |

:::note
지원되지 않는 C++ 문법(템플릿, 가상 함수 등)이 포함된 경우 파싱 오류가 발생할 수 있습니다. 순수 데이터 구조체만 사용하세요.
:::
