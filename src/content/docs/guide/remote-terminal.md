---
title: 원격 터미널 (SSH/RDP)
description: 웹 브라우저에서 SSH/RDP로 테스트 VM에 원격 접속하는 방법
---

원격 터미널은 웹 브라우저에서 별도의 SSH 클라이언트 없이 테스트 VM에 직접 접속할 수 있는 기능입니다. 페이지 경로: `/remote`

## 개요

xterm.js 기반 SSH 터미널로 브라우저에서 직접 VM에 접속합니다. VPN이나 별도 소프트웨어 설치 없이 웹 브라우저만으로 원격 접속이 가능합니다.

- **텍스트 선택**: 마우스 드래그로 자연스럽게 선택
- **복사**: Cmd+C (선택 텍스트가 있으면 복사, 없으면 SIGINT)
- **붙여넣기**: Cmd+V (브라우저 네이티브)

## VM 선택

접속할 VM을 선택합니다. 사용 가능한 VM 목록이 표시됩니다:

- **T1 ~ T4** — Tentacle 서버 (테스트 실행 환경)
- **HEAD** — Head 서버 (중앙 제어)

## 다중 터미널 탭

여러 VM에 동시에 접속할 수 있습니다. 각 접속은 별도의 탭으로 관리됩니다.

- 탭 클릭으로 터미널 간 전환
- 탭의 X 버튼으로 접속 종료
- 필요한 만큼 새 탭 추가 가능

## Broadcast 명령어

**Broadcast** 기능을 사용하면 열려 있는 **모든 터미널 탭에 동시에 동일한 명령어**를 전송할 수 있습니다.

여러 VM에 같은 명령을 실행해야 할 때 유용합니다. 예를 들어:

- 모든 Tentacle에서 로그 확인
- 일괄 서비스 재시작
- 동시 상태 점검

:::tip
Broadcast는 SSH 터미널에서만 동작합니다. RDP 탭에는 영향이 없습니다.
:::

## 슬롯 페이지에서 터미널 열기

Slots 페이지에서 슬롯을 우클릭한 후 **Terminal** 메뉴를 선택하면, 해당 슬롯이 연결된 VM으로 바로 원격 터미널이 열립니다. 별도로 VM을 선택할 필요가 없습니다.

## 전체화면 토글

터미널 영역을 전체화면으로 확장할 수 있습니다. 전체화면 버튼을 클릭하거나 단축키를 사용합니다. 다시 클릭하면 원래 크기로 복원됩니다.

## 세션 잠금 (Session Locking)

RDP/VNC 원격 접속 시 한 VM에 대해 **한 명만 동시 접속**할 수 있도록 세션 잠금이 적용됩니다. 이는 여러 사용자가 동일한 VM을 동시에 조작하여 충돌이 발생하는 것을 방지합니다.

### SessionLockManager

`SessionLockManager`는 VM별 독점 접근을 관리하는 서버 컴포넌트입니다.

- **Lock 획득**: RDP/VNC 연결 시도 시 자동으로 Lock 획득을 시도합니다. 같은 사용자가 재접속하면 기존 Lock이 갱신됩니다.
- **Lock 차단**: 다른 사용자가 이미 사용 중인 VM에 접속하려 하면, 현재 Lock을 보유한 사용자 정보와 함께 접속이 거부됩니다.
- **Lock 해제**: 정상적으로 연결을 종료하면 Lock이 자동으로 해제됩니다.

### Heartbeat 메커니즘

비정상 종료(브라우저 충돌, 네트워크 끊김 등)로 인해 Lock이 해제되지 않는 상황을 방지하기 위해 Heartbeat 메커니즘이 적용됩니다.

- 연결된 클라이언트는 주기적으로 Heartbeat 신호를 서버에 전송합니다.
- Heartbeat가 **5분간 수신되지 않으면** 해당 세션의 Lock이 만료된 것으로 간주합니다.
- 만료된 Lock은 다른 사용자의 접속 시도 시 자동으로 해제되며, 새 사용자에게 Lock이 부여됩니다.
- 서버는 1분마다 만료된 Lock을 스캔하여 자동 정리합니다 (`@Scheduled(fixedRate = 60_000)`).

### 관리자 강제 해제

관리자는 특정 VM의 세션 Lock을 강제로 해제할 수 있습니다. 이는 사용자가 오랫동안 접속을 유지하고 있거나, 비정상적인 상황에서 Lock이 해제되지 않을 때 사용합니다.

- 강제 해제 시 해당 VM에 대한 모든 Lock 정보와 접속 시도 기록이 함께 삭제됩니다.
- 현재 연결 중인 세션이 있더라도 Lock이 해제되므로, 다른 사용자가 새로 접속할 수 있게 됩니다.

### 접속 시도 알림

다른 사용자가 내가 사용 중인 VM에 접속을 시도하면, 해당 시도가 기록됩니다.

- 현재 Lock 보유자는 누가 접속을 시도했는지 확인할 수 있습니다 (사용자명, 시도 시각).
- 알림 확인 후 기록은 자동으로 삭제됩니다 (poll 방식).
- 이를 통해 다른 팀원이 해당 VM을 사용하려고 대기하고 있는지 파악하고, 필요 시 연결을 종료하여 양보할 수 있습니다.

:::caution
세션 잠금은 RDP/VNC 접속에만 적용됩니다. SSH 터미널은 여러 사용자가 동시에 접속할 수 있습니다.
:::
