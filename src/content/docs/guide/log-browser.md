로그 브라우저는 테스트 실행 중 생성된 로그 파일을 웹에서 탐색하고 조회할 수 있는 기능입니다. 별도의 SSH 접속 없이 파일을 찾고, 내용을 확인하고, 다운로드할 수 있습니다.

## 로그 브라우저 — 파일 탐색

### 디렉토리 트리 탐색

서버의 로그 디렉토리를 트리 형태로 탐색합니다. 폴더를 클릭하여 하위 디렉토리로 이동하고, 상위 경로 버튼으로 돌아갈 수 있습니다.

### 파일 정보 표시

각 파일/폴더에 대해 다음 정보가 표시됩니다:

| 정보 | 설명 |
|------|------|
| 이름 | 파일/폴더명 |
| 크기 | 파일 크기 (사람이 읽기 쉬운 형식) |
| 수정일 | 마지막 수정 시각 |

### 파일 다운로드

파일 목록에서 다운로드 버튼을 클릭하여 로그 파일을 로컬에 다운로드할 수 있습니다.

## 로그 뷰어 — 파일 내용 조회

파일을 클릭하면 로그 뷰어가 열려 파일 내용을 확인할 수 있습니다.

### 청크 로딩

대용량 로그 파일은 **1000줄씩 청크 단위**로 로딩됩니다. 스크롤하거나 페이지 버튼을 클릭하여 다음 청크를 불러옵니다.

### Last 버튼

**Last** 버튼을 클릭하면 파일의 마지막 **2000줄**을 바로 표시합니다. 실시간으로 기록되는 로그의 최신 내용을 빠르게 확인할 때 유용합니다.

### ripgrep 패턴 검색

로그 뷰어 상단의 검색 필드에 ripgrep 정규식 패턴을 입력하여 파일 내용을 검색할 수 있습니다.

- 최대 **500건**까지 검색 결과 표시
- 정규식 패턴 지원 (예: `error|fail`, `timeout \d+ms`)
- 매칭된 줄이 하이라이트 표시

:::tip
ripgrep 패턴은 대소문자를 구분합니다. 대소문자 무시 검색이 필요하면 `(?i)` 접두사를 사용하세요. 예: `(?i)error`
:::

### 바이너리 파일 감지

바이너리 파일을 열려고 하면 자동으로 감지하여 경고 메시지를 표시합니다. 바이너리 파일은 텍스트 뷰어로 조회할 수 없습니다.

### non-UTF-8 인코딩 자동 변환

EUC-KR 등 UTF-8이 아닌 인코딩의 파일도 자동으로 변환하여 올바르게 표시합니다.

## 접속 모드

| 모드 | 설명 |
|------|------|
| **SSH** | SSH를 통해 원격 서버의 로그에 접근 (기본 모드) |
| **Local** | 마운트된 로컬 경로에서 로그 파일에 접근 |

:::note
접속 모드는 서버 설정(`application.yaml`)에 따라 결정됩니다. 일반 사용자는 별도로 설정할 필요가 없습니다.
:::

## 사용 컨텍스트

로그 브라우저는 다음 경로에서 접근할 수 있습니다:

- **History 결과 페이지** — 테스트 결과의 로그 경로를 클릭하여 해당 로그 파일 직접 조회
- **Slots 페이지** — 슬롯의 Context Menu에서 로그 탐색
