---
title: Pre-Command (사전 명령어)
description: 슬롯/TC 단위로 테스트 시작 전 adb/shell 명령어를 자동 실행하는 Pre-Command 기능의 설계, 구현, 사용법, UX 가이드
---

Pre-Command는 테스트 시작 전에 슬롯의 디바이스에 필요한 명령어(adb push, chmod 등)를 미리 실행하는 기능입니다. 명령어 템플릿을 만들어 두고, **슬롯 단위** 또는 **TC 단위**로 등록하면 init 상태 진입 시 자동으로 실행됩니다.

---

## 1. 개요

### 두 가지 등록 레벨

Pre-Command는 두 가지 레벨로 등록할 수 있습니다.

| | 슬롯 Pre-Command | TC Pre-Command |
|---|---|---|
| **등록 단위** | 슬롯 1개당 1개 | TC별로 각각 등록 |
| **등록 위치** | Pre-Command 시트 | TC 테이블의 Pre-Cmd 컬럼 |
| **실행 조건** | init 진입 시 항상 | init 진입 시 + 현재 TC 매칭 |
| **우선순위** | 낮음 | **높음** (TC 것이 있으면 슬롯 것 무시) |
| **삭제** | 수동 해제 | 슬롯 clear 시 자동 삭제 |

### 우선순위 규칙

슬롯이 init 상태에 진입하면 다음 순서로 실행할 명령어를 결정합니다:

1. 현재 TC (`currentRunningTc`)에 Pre-Command가 등록되어 있는지 확인
2. **있으면** → TC Pre-Command 실행 (슬롯 Pre-Command 건너뜀)
3. **없으면** → 슬롯 Pre-Command 실행 (기존 동작)

:::tip
TC Pre-Command가 하나라도 등록되면 해당 슬롯의 슬롯 Pre-Command는 **자동 해제**됩니다. TC 단위 제어와 슬롯 단위 제어가 혼합되지 않도록 설계되었습니다.
:::

## 2. 접근 방법

슬롯 페이지에서 슬롯을 선택한 후 다음 방법으로 접근합니다:

| 방법 | 경로 |
|------|------|
| **컨텍스트 메뉴** | 슬롯 우클릭 → Prepare Test → Pre-Command |
| **선택 시트** | 슬롯 다중 선택 → 하단 "Pre-Command" 버튼 |

---

## 3. 명령어 템플릿 관리

### 통합 시트

Pre-Command를 클릭하면 우측 시트가 열리며, 세 가지 뷰를 제공합니다. 한 화면에서 등록/실행/관리를 모두 처리할 수 있습니다.

#### 메인 뷰

선택된 슬롯과 명령어 템플릿 목록이 표시됩니다.

| 요소 | 설명 |
|------|------|
| **선택된 슬롯** | 상단에 현재 선택된 슬롯 이름 표시 (3개 초과 시 "외 N개" 축약) |
| **등록 상태** | 현재 등록된 템플릿에 ✅ 체크 + primary 색상 강조 |
| **혼합 표시** | 슬롯별 등록이 다르면 amber 경고 메시지 |
| **등록/해제 버튼** | 각 템플릿 카드에 상태에 따라 "등록" 또는 "해제" 버튼 |
| **즉시 실행** | 모든 템플릿에 ▶ "즉시 실행" 버튼 |
| **관리 링크** | 하단 ⚙ "관리" 버튼으로 관리 뷰 전환 |
| **전체 해제** | 하단 우측, 등록된 상태에서만 표시 |

:::tip
템플릿이 하나도 없으면 예시 명령어(adb push / chmod)와 함께 "첫 명령어 만들기" 버튼이 표시됩니다. 예시를 참고해서 빠르게 시작할 수 있습니다.
:::

#### 관리 뷰

명령어 템플릿의 편집/삭제를 수행합니다.

- 각 카드에 hover 시 편집(✏️)/삭제(🗑️) 아이콘 표시
- hover 시 테두리 강조 + 그림자 효과
- 하단 "새 명령어 추가" 버튼으로 생성
- ← 뒤로가기로 메인 뷰 복귀

#### 편집 뷰

명령어 템플릿을 생성하거나 수정합니다.

| 필드 | 설명 |
|------|------|
| **이름** | 템플릿 이름 (예: "tiotest 설치") |
| **설명** | 선택 사항. 용도 메모 |
| **명령어** | 한 줄에 하나씩 입력. adb 명령어는 자동으로 `-s {usbId}` 삽입 |

**명령어 입력 예시:**
```
adb push tiotest-0.52 /dev
adb shell chmod +x /dev/tiotest-0.52
adb shell ls -al /dev/tiotest*
```

## 4. 슬롯 Pre-Command 등록 (자동 실행)

슬롯에 Pre-Command를 등록하면, 해당 슬롯이 **init 상태에 진입할 때 자동으로 실행**됩니다.

### 등록 방법

1. 슬롯 선택 → Pre-Command 시트 열기
2. 원하는 템플릿의 "등록" 버튼 클릭
3. 슬롯 카드 헤더에 ⚡ 아이콘 표시됨

### 해제 방법

- 시트에서 등록된 템플릿의 "해제" 버튼 클릭
- 또는 하단 "전체 해제" 버튼

### 자동 실행 조건

| 조건 | 설명 |
|------|------|
| 상태 변화 | 슬롯의 `testState`가 init을 포함하는 값으로 변경될 때 |
| 이전 상태 | 이전 상태가 init이 아니어야 함 (이미 init이면 무시) |
| 중복 방지 | 같은 슬롯에 대해 한 번만 실행, 다른 상태로 갔다가 다시 init이 되면 재실행 |
| TC 우선순위 | 현재 TC에 TC Pre-Command가 등록되어 있으면 슬롯 Pre-Command는 건너뜀 |

### 슬롯 카드 시각적 피드백

| 요소 | 설명 |
|------|------|
| **⚡ 뱃지** | 카드 헤더에 번개 아이콘 (메모 아이콘 옆) |
| **툴팁** | 카드 hover 시 "Pre-Cmd: tiotest 설치" 등 등록된 템플릿 이름 표시 |

---

## 5. TC Pre-Command 등록

TC 단위로 Pre-Command를 등록하면, 해당 TC가 실행될 때 **슬롯 Pre-Command 대신** TC Pre-Command가 실행됩니다.

### 등록 방법

슬롯 모니터링 페이지에서:

1. 슬롯 선택 → 우측 TC 테이블 표시
2. 각 TC 행의 **Pre-Cmd** 컬럼에 드롭다운 표시
3. 드롭다운에서 Pre-Command 선택 → **즉시 DB 등록**
4. X 버튼으로 해제

:::caution
**NOTSTART** 상태인 TC만 Pre-Command를 설정/변경할 수 있습니다. 실행 중이거나 완료된 TC는 읽기 전용입니다.
:::

### TC Pre-Command 등록 시 동작

- 해당 슬롯의 **슬롯 Pre-Command가 자동 해제**됩니다
- TC별로 다른 Pre-Command를 설정할 수 있습니다
- 같은 슬롯 내 TC A는 "tiotest 설치", TC B는 "fio 설치" 등 개별 설정 가능

### 자동 삭제

TC Pre-Command는 다음 상황에서 자동 삭제됩니다:

| 상황 | 삭제 범위 |
|------|-----------|
| **슬롯 clear** | 해당 슬롯의 **모든** TC Pre-Command |
| **TC 삭제** (deleteTest) | 해당 TC의 Pre-Command |
| **수동 해제** (X 버튼) | 해당 TC의 Pre-Command |

---

## 6. 즉시 실행

등록 없이 선택한 슬롯들에 바로 명령어를 실행합니다.

1. 슬롯 선택 → Pre-Command 시트 열기
2. 원하는 템플릿의 "즉시 실행" 버튼 클릭
3. 우하단 플로팅 카드에서 실시간 진행 표시

:::tip
즉시 실행은 슬롯 상태에 관계없이 실행 가능합니다. init 상태가 아니어도 됩니다.
:::

## 7. 플로팅 카드 (실행 진행)

명령어 실행 중 우하단에 플로팅 카드가 표시됩니다. 닫기 전까지 계속 유지되며, 슬롯 페이지의 다른 작업을 방해하지 않습니다.

### 프로그레스 바

| 상태 | 표시 |
|------|------|
| **실행 중** | 파란색 바가 진행률에 따라 채워짐 (접었을 때/펼쳤을 때 모두 표시) |
| **성공 완료** | 초록색 풀 바 |
| **실패 포함** | 빨간색 풀 바 |

### 표시 정보

| 요소 | 설명 |
|------|------|
| **헤더** | 템플릿 이름 + 완료/전체 카운트 |
| **상태 아이콘** | 🔄 실행 중 / ✅ 성공 / ❌ 실패 / ⏭️ 스킵 |
| **슬롯별 진행** | 각 슬롯의 명령어 실행 상태 (아코디언) |
| **명령어 출력** | 성공 시 stdout, 실패 시 stderr 표시 |
| **접기/펼치기** | 헤더 클릭으로 카드 접기 |

### 자동 동작

| 동작 | 설명 |
|------|------|
| **실행 중 슬롯 자동 펼침** | 현재 실행 중인 슬롯이 자동으로 펼쳐짐 |
| **실패 슬롯 자동 펼침** | 완료 후 실패한 슬롯이 자동으로 펼쳐져 에러를 즉시 확인 |
| **완료 토스트** | 카드 닫을 때 "tiotest 설치: 3 성공 · 1 실패" 요약 토스트 |

### 슬롯 스킵 사유

서버에서 슬롯 검증 실패 시 스킵 사유가 표시됩니다:
- "슬롯 데이터 없음"
- "서버 정보 추출 실패"

## 8. adb 명령어 자동 처리

`adb`로 시작하는 명령어에는 자동으로 `-s {usbId}`가 삽입됩니다.

| 입력 | 실제 실행 |
|------|-----------|
| `adb push file /dev` | `adb -s usb:9-1.4.1 push file /dev` |
| `adb shell chmod +x /dev/file` | `adb -s usb:9-1.4.1 shell chmod +x /dev/file` |
| `ls -al /data` | `ls -al /data` (adb 아님, 변환 없음) |

usbId는 Head 슬롯 데이터에서 실시간으로 가져옵니다.

## 9. SSH 실행 환경

명령어는 슬롯이 연결된 Tentacle 서버에 SSH로 접속하여 실행됩니다.

| 항목 | 설명 |
|------|------|
| **SSH 대상** | 슬롯의 `setLocation`에서 추출 (예: "T1-S03" → T1 서버) |
| **접속 정보** | `portal_servers` 테이블의 SSH 설정 (ip, username, password, ssh_port) |
| **실행 방식** | JSch `ChannelExec` (단발 명령어 실행, 터미널 아님) |
| **타임아웃** | 명령어당 60초. 초과 시 `[TIMEOUT]` 메시지와 함께 실패 |
| **실패 처리** | 명령어 실패(exit code ≠ 0) 시 해당 슬롯의 나머지 명령어 중단 |
| **동시 실행** | 즉시 실행 최대 8개, 자동 실행 최대 4개 동시 처리 (스레드풀 제한) |

## 10. 전체 워크플로우 요약

### 시나리오 1: 슬롯 전체에 동일 명령어

모든 TC가 같은 준비 명령어를 필요로 하는 경우:

1. Pre-Command 시트에서 "tiotest 설치" 템플릿 생성
2. 슬롯 선택 → "등록" 클릭
3. 이후 모든 init마다 자동 실행

### 시나리오 2: TC별로 다른 명령어

TC A는 tiotest, TC B는 fio가 필요한 경우:

1. 두 템플릿("tiotest 설치", "fio 설치") 생성
2. TC 테이블에서 TC A → "tiotest 설치", TC B → "fio 설치" 각각 등록
3. 슬롯 Pre-Command는 자동 해제됨
4. TC A init → tiotest 설치 실행, TC B init → fio 설치 실행

### 시나리오 3: 즉시 실행 (일회성)

등록 없이 지금 바로 실행하고 싶은 경우:

1. 슬롯 선택 → Pre-Command 시트 열기
2. "즉시 실행" 클릭 → 플로팅 카드에서 결과 확인

## 11. 활용 예시

### tiotest 바이너리 배포

```
adb push tiotest-0.52 /dev
adb shell chmod +x /dev/tiotest-0.52
```

### fio 설치 및 확인

```
adb push fio /dev
adb shell chmod +x /dev/fio
adb shell /dev/fio --version
```

### 테스트 디렉토리 정리

```
adb shell rm -rf /data/test_output
adb shell mkdir -p /data/test_output
```

### 디바이스 정보 수집

```
adb shell cat /proc/version
adb shell df -h /data
adb shell getprop ro.build.display.id
```
